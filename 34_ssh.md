# SSH

## Вопрос 1. Что такое SSH и для чего он нужен?

**Ответ:** SSH — это и протокол для удаленного подключения к другим хостам, и утилита в Linux.

**Дополнение:**
- **Протокол** — набор правил для защищенного удаленного управления и передачи данных
- **Утилита** — команда `ssh` для подключения к удаленным хостам

**Основные возможности:**
- Удаленный доступ к терминалу
- Безопасная передача файлов (scp, sftp)
- Туннелирование/проброс портов
- Выполнение команд без интерактивной сессии

**Важно:** В отличие от Telnet, SSH **шифрует** все данные (пароли, команды, файлы).

---

## Вопрос 2. На каком порту работает SSH?

**Ответ:** На 22 порту.

**Дополнение:**
- По умолчанию — TCP порт 22
- Можно изменить в файле `/etc/ssh/sshd_config` (строка `Port`)

## Вопрос 3. В чем отличие публичного и приватного ключа?

**Ответ:** Публичный ключ передается хосту, к которому ты подключаешься (он есть и у тебя, и на сервере). Приватный ключ есть только у тебя. Сервер отправляет сообщение, ты подписываешь его приватным ключом и отправляешь обратно. Сервер с помощью публичного ключа проверяет подпись — если всё ок, подключает.

**Дополнение (важное уточнение):**
- **Приватный ключ** — только у владельца, используется для **подписи** (доказательства личности)
- **Публичный ключ** — можно распространять, используется для **проверки** подписи
- Приватным ключом **не шифруют** — он создает подпись. Публичный ключ эту подпись проверяет
- Математически из публичного ключа **невозможно** вычислить приватный

---

## Вопрос 4. Можно ли передавать свой открытый ключ публично?

**Ответ:** Да, главное — не передавать приватный.

**Дополнение:**
- Публичный ключ **можно и нужно** передавать публично
- Его можно класть в `authorized_keys` на серверах, отправлять коллегам, публиковать на GitHub
- Из публичного ключа невозможно вычислить приватный
- **Приватный ключ нельзя передавать НИКОМУ и НИКОГДА**

---

## Вопрос 5. Позволяет ли SSH выполнять команды или копировать файлы без явного подключения к серверу?

**Ответ:** Да.

**Дополнение:**

**Выполнение команд без интерактивной сессии:**
```bash
ssh user@host ls -la
ssh user@host "df -h"
ssh user@host "sudo systemctl restart nginx"
```

**Копирование файлов:**
```bash
scp file.txt user@host:/path/           # копирование на сервер
scp user@host:/remote/file.txt ./       # копирование с сервера
rsync -avz ./local/ user@host:/remote/  # синхронизация
```

| Тип подключения | Команда | Интерактивная сессия? |
|-----------------|---------|----------------------|
| Полная сессия | `ssh user@host` | Да |
| Выполнение команды | `ssh user@host ls` | Нет |
| Копирование файлов | `scp`, `rsync` | Нет |
