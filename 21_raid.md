## Программный RAID


## Вопросы и ответы

### Вопрос 1: Зачем нужен RAID?

**Ответ:** RAID (Redundant Array of Independent Disks) нужен для:

| Цель | Объяснение |
|------|------------|
| **Отказоустойчивость** | при выходе одного диска из строя данные сохраняются |
| **Доступность** | система продолжает работать в режиме degraded |
| **Объединение дисков** | несколько дисков работают как одно устройство |
| **Повышение производительности** | некоторые уровни RAID увеличивают скорость чтения/записи |

>  RAID защищает от **аппаратных проблем** (отказ диска), но не заменяет бэкап.

---

### Вопрос 2: Какие уровни рейда вы знаете и чем они отличаются?

**Ответ:** основные уровни RAID:

| Уровень | Название | Минимум дисков | Отказоустойчивость | Полезный объём | Скорость |
|---------|----------|----------------|-------------------|----------------|----------|
| **RAID 0** | Striping (чередование) | 2 | нет | сумма всех дисков | высокая |
| **RAID 1** | Mirror (зеркало) | 2 | 1 диск | размер одного диска | средняя |
| **RAID 5** | Parity (чётность) | 3 | 1 диск | (N-1) × размер | высокая (чтение) |
| **RAID 6** | Double Parity | 4 | 2 диска | (N-2) × размер | высокая (чтение) |
| **RAID 10** | Mirror + Striping | 4 | до 2 дисков | (N/2) × размер | высокая |

#### Подробнее о каждом:

| Уровень | Плюсы | Минусы | Применение |
|---------|-------|--------|------------|
| **RAID 0** | максимальная скорость, полный объём | нет отказоустойчивости | видеомонтаж, временные данные |
| **RAID 1** | отказоустойчивость, простота | полезный объём = 1 диск | ОС, базы данных |
| **RAID 5** | баланс цены и надёжности | медленная запись | файловые серверы |
| **RAID 6** | выдерживает отказ 2 дисков | ещё медленнее запись | большие хранилища |
| **RAID 10** | скорость + надёжность | дорого (нужно 4 диска) | высоконагруженные серверы |

---

### Вопрос 3: Чем отличается программный рейд от аппаратного?

**Ответ:**

| Характеристика | Программный RAID | Аппаратный RAID |
|----------------|------------------|-----------------|
| **Где работает** | на уровне ОС (через ядро) | на отдельном контроллере |
| **Использует CPU** | да | нет |
| **Стоимость** | бесплатно | дорого (нужен контроллер) |
| **Зависимость от ОС** | да | нет |
| **Переносимость** | высокая (любой Linux) | низкая (нужен тот же контроллер) |
| **Скорость** | ниже (CPU занят) | выше (свои ресурсы) |
| **Батарея для кэша** | нет | может быть |

---

### Вопрос 4: Заменяет ли рейд бэкап и почему?

**Ответ:** НЕТ, RAID **не заменяет** бэкап.

| Сценарий | RAID защищает? | Бэкап защищает? |
|----------|----------------|-----------------|
| Отказ одного диска | да | да |
| Случайное удаление файла | нет (удалится на всех дисках) | да |
| Вирус или шифровальщик | нет | да |
| Пожар, кража, наводнение | нет | да (если бэкап в другом месте) |
| Ошибка пользователя | нет | да |

**Правило:**
> **RAID — защита от аппаратных проблем. Бэкап — защита от логических проблем.**
> 
> RAID + бэкап = правильная стратегия.

---

### Вопрос 5: Как посмотреть информацию о RAID устройстве? Как понять, есть ли проблемы?

**Ответ:** с помощью команд `mdadm -D` и `cat /proc/mdstat`.

| Команда | Что показывает |
|---------|----------------|
| `sudo mdadm -D /dev/mdX` | детальная информация о RAID |
| `cat /proc/mdstat` | краткое состояние всех RAID |
| `sudo mdadm --detail --scan` | информация для конфигурации |

#### Как понять, что есть проблемы:

| Признак проблемы | Что искать |
|------------------|------------|
| **degraded** | один диск отсутствует или сломан |
| **removed** | диск удалён из массива |
| **failed** | диск помечен как неисправный |
| **spare rebuilding** | идёт восстановление (не проблема, а процесс) |
| `[U_]` или `[_U]` в `/proc/mdstat` | один диск работает, второй отсутствует |
| `[UU]` в `/proc/mdstat` | оба диска работают нормально |

## Практическое задание

### Условие задания

Создайте RAID 1 из двух разделов на разных дисках, поверх RAID настройте LVM с одним логическим разделом, который должен занимать **60% пространства группы**, а файловая система должна монтироваться в директорию `/srv`. Выключите виртуалку, удалите один из дисков, добавьте новый диск и восстановите RAID.

---

### Выполнение задания

#### Шаг 1: Подготовка дисков

```bash
# Просмотр доступных дисков
lsblk

# Очистка дисков от старых меток (если диски использовались ранее)
sudo wipefs -a /dev/sdc /dev/sde
sudo mdadm --zero-superblock /dev/sdc1 /dev/sde1 2>/dev/null
```

#### Шаг 2: Создание разделов на двух дисках

```bash
# На первом диске
sudo fdisk /dev/sdc
# g → создать GPT таблицу
# n → Enter → Enter → +1G → создать раздел 1GB
# w

# На втором диске
sudo fdisk /dev/sde
# g → создать GPT таблицу
# n → Enter → Enter → +1G → создать раздел 1GB
# w
```

**Результат:**
```
/dev/sdc1   954M   part
/dev/sde1   954M   part
```

#### Шаг 3: Создание RAID 1

```bash
sudo mdadm --create /dev/md/raid_t --level=1 --raid-devices=2 /dev/sdc1 /dev/sde1
# подтвердить создание (y)
```

**Проверка:**
```bash
cat /proc/mdstat
# [UU] — оба диска работают
sudo mdadm -D /dev/md/raid_t
```

#### Шаг 4: Сохранение конфигурации RAID

```bash
echo 'DEVICE /dev/sd*[0-9]' | sudo tee /etc/mdadm.conf
sudo mdadm --detail --scan | sudo tee -a /etc/mdadm.conf
```

#### Шаг 5: Создание LVM поверх RAID

```bash
# Создание физического тома
sudo pvcreate /dev/md/raid_t

# Создание группы томов
sudo vgcreate raid_vg /dev/md/raid_t

# Создание логического тома на 60% VG
sudo lvcreate -l 60%VG -n raid_lv raid_vg

# Создание файловой системы ext4
sudo mkfs.ext4 /dev/raid_vg/raid_lv
```

#### Шаг 6: Монтирование в /srv

```bash
# Создание точки монтирования
sudo mkdir -p /srv

# Монтирование
sudo mount /dev/raid_vg/raid_lv /srv

# Добавление в fstab для автомонтирования
echo "/dev/raid_vg/raid_lv /srv ext4 defaults 0 0" | sudo tee -a /etc/fstab

# Проверка
df -h /srv
lsblk
```

**Схема после настройки:**
```
Диск 1 (/dev/sdc1) ─┐
                     ├→ RAID 1 (/dev/md/raid_t) → PV → VG (raid_vg) → LV (raid_lv) → /srv (ext4)
Диск 2 (/dev/sde1) ─┘
```

#### Шаг 7: Имитация отказа диска

```bash
# Выключение виртуальной машины
sudo shutdown -h now
```

В VirtualBox:
- Настройки ВМ → Носители
- Удалить один из дисков (например, `/dev/sdc`)
- Включить ВМ

**Проверка состояния после загрузки:**
```bash
sudo mdadm -D /dev/md/raid_t
# State : clean, degraded
# один диск removed

cat /proc/mdstat
# [U_] или [_U] — один диск работает

# Данные в /srv доступны
ls /srv
```

#### Шаг 8: Восстановление RAID новым диском

```bash
# Выключение виртуальной машины
sudo shutdown -h now
```

В VirtualBox:
- Добавить новый диск (например, `/dev/sdf`, 1GB)
- Включить ВМ

```bash
# Создание раздела на новом диске
sudo fdisk /dev/sdf
# g → n → +1G → w

# Очистка суперблока (на всякий случай)
sudo mdadm --zero-superblock /dev/sdf1

# Добавление нового раздела в RAID
sudo mdadm /dev/md/raid_t --add /dev/sdf1

# Наблюдение за процессом восстановления
watch -n 1 cat /proc/mdstat
# через некоторое время: [UU]
```

#### Шаг 9: Финальная проверка

```bash
# Проверка состояния RAID
sudo mdadm -D /dev/md/raid_t
# State : clean
# Active Devices : 2

# Проверка LVM
sudo lvs
sudo vgs
sudo pvs

# Проверка монтирования
df -h /srv
```

### Основные команды

| Команда | Что делает |
|---------|------------|
| `mdadm --create /dev/md/name --level=1 --raid-devices=2 /dev/sdX1 /dev/sdY1` | создать RAID 1 |
| `cat /proc/mdstat` | просмотр состояния RAID |
| `mdadm -D /dev/md/name` | детальная информация о RAID |
| `mdadm /dev/md/name --add /dev/sdZ1` | добавить диск в RAID |
| `mdadm /dev/md/name --fail /dev/sdX1` | пометить диск как сломанный |
| `mdadm /dev/md/name --remove /dev/sdX1` | удалить диск из RAID |
| `mdadm --stop /dev/md/name` | остановить RAID |
| `mdadm --zero-superblock /dev/sdX1` | удалить суперблок RAID |
| `mdadm --detail --scan >> /etc/mdadm.conf` | сохранить конфигурацию |

### Специальные биты в `/proc/mdstat`

| Символ | Значение |
|--------|----------|
| `[UU]` | оба диска работают нормально |
| `[U_]` | первый диск работает, второй отсутствует |
| `[_U]` | первый диск отсутствует, второй работает |
| `[U U]` | идёт восстановление (spare rebuilding) |
