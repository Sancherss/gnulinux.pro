# sudo

## Вопросы

### Вопрос 1: Чем отличается su от sudo?

| | `su` | `sudo` |
|--|------|--------|
| **Что делает** | переключает на другого пользователя | выполняет команду от имени другого пользователя |
| **Нужен ли пароль** | пароль **целевого** пользователя | **свой** пароль |
| **Как работает** | создаёт новую сессию | выполняет одну команду |
| **Логирование** | не логирует команды | логирует все команды |
| **Гибкость** | все или ничего | можно дать права на конкретные команды |

**Примеры:**
```bash
# su — переключение на root
su root
# нужен пароль root

# sudo — выполнить одну команду от root
sudo cat /etc/shadow
# нужен свой пароль
```

---

### Вопрос 2: Кто может использовать sudo?

**Ответ:** всё зависит от настроек в `/etc/sudoers` (и `/etc/sudoers.d/*`).

**По умолчанию:**
- `wheel` (CentOS, RHEL, Fedora)
- `sudo` (Ubuntu, Debian)

**Примеры настроек:**
| Запись | Кто может использовать sudo |
|--------|----------------------------|
| `root ALL=(ALL) ALL` | только root |
| `%wheel ALL=(ALL) ALL` | все пользователи группы wheel |
| `sancher ALL=(ALL) ALL` | только пользователь sancher |
| `ALL ALL=(ALL) ALL` | **все** пользователи (опасно) |

### Вопрос 3: Чем отличается `su user` от `su - user`?

| | `su user` | `su - user` |
|--|-----------|-------------|
| **Окружение** | остаётся от старого пользователя | как при логине (новое) |
| **Домашняя директория** | остаётся в старой | переходит в `/home/user` |
| **Файлы профиля** | не читает `.bash_profile` | читает `.bash_profile` |
| **Переменные окружения** | старые | новые |

**Пример:**
```bash
[root@centos8 tmp]# su sancher
[sancher@centos8 tmp]$ pwd
/tmp   # директория не изменилась

[root@centos8 tmp]# su - sancher
[sancher@centos8 ~]$ pwd
/home/sancher   # перешёл в домашнюю папку
```

### Вопрос 4: Чем отличается `su` от `sudo su`?

| | `su` | `sudo su` |
|--|------|-----------|
| **Какой пароль нужен** | пароль **root** | **свой** пароль |
| **Кто может использовать** | только те, кто знает пароль root | только те, у кого есть sudo |
| **Логирование** | не логируется | логируется |

```bash
sancher@centos8:~$ su
Password: (нужен пароль root)

sancher@centos8:~$ sudo su
[sudo] password for sancher: (свой пароль)
```

> **Лучший способ получить root:** `sudo -i`

### Вопрос 5: Объясните запись `ALL=(ALL:ALL) ALL`

```
ALL = ( ALL : ALL ) ALL
│      │     │      │
│      │     │      └── какие команды: все
│      │     └───────── от каких групп: от всех
│      └─────────────── от имени каких пользователей: от всех
└────────────────────── какие пользователи: все
```

**Значение:** любой пользователь может выполнить любую команду от имени любого пользователя и любой группы.

### Вопрос 6: Как дать пользователю права администратора?

**Способы:**

| Способ | Команда | Плюсы | Минусы |
|--------|---------|-------|--------|
| Добавить в группу | `usermod -aG wheel user` | просто, легко отозвать | даёт **все** права |
| Через sudoers | `user ALL=(ALL) /usr/bin/systemctl` | **ограниченные** права | сложнее настраивать |

**Лучший способ:** группа + ограничения через sudoers

```bash
# 1. Добавить в группу wheel
sudo usermod -aG wheel user2

# 2. Ограничить команды в sudoers
user2 ALL=(ALL) /usr/bin/systemctl, /usr/bin/journalctl
```

> **Принцип наименьших привилегий:** давай ровно столько прав, сколько нужно.

## Задание 1: Создание администратора user2

### Выполнение

```bash
# 1. Создание пользователя
sudo useradd user2
sudo passwd user2

# 2. Настройка sudoers
echo "user2 ALL=(ALL:ALL) ALL" | sudo tee /etc/sudoers.d/user2_file
```

### Проверка прав

```bash
su - user2
sudo -l
# User user2 may run the following commands on centos8:
#    (ALL : ALL) ALL
```

### Доказательство

| Действие | Команда | Результат |
|----------|---------|-----------|
| Чтение /etc/passwd | `sudo cat /etc/passwd` | успешно |
| Создание пользователя | `sudo useradd user3` | успешно |
| Удаление пользователя | `sudo userdel user3` | успешно |

**Вывод:** user2 имеет полные права администратора.

## Задание 2: user1 запускает touch от user2

### Настройка sudoers

Создан файл `/etc/sudoers.d/user1_file`:

```bash
user1 ALL=(user2) /bin/touch
```

### Проверка

```bash
# Переключился на user1
su - user1

# Перешёл в /tmp (где user2 имеет права на запись)
cd /tmp

# Выполнил touch от имени user2
sudo -u user2 touch file

# Проверил результат
ls -l file
# -rw-r--r-- 1 user2 user2 0 апр 18 11:09 file
```

### Почему не сработало в домашней папке?

`sudo -u user2 touch file` в `/home/user1` не работал, потому что user2 не имеет прав на запись в чужую домашнюю директорию.  
В `/tmp` — работает, потому что `/tmp` доступен всем пользователям на запись.

### Вывод

| Критерий | Результат |
|----------|-----------|
| user1 может выполнять `touch` от user2 | да |
| Файл создаётся с владельцем user2 | да |

### Формат записи

```
пользователь  хост=(пользователь:группа)  команды
```

### Примеры

| Запись | Значение |
|--------|----------|
| `user ALL=(ALL) ALL` | полные права |
| `user ALL=(ALL) /bin/touch` | только команда touch |
| `user ALL=(user2) /bin/touch` | touch только от user2 |
| `%wheel ALL=(ALL) ALL` | вся группа wheel |
| `user ALL=(ALL) NOPASSWD: /bin/touch` | без запроса пароля |

### Где хранятся настройки

| Файл | Назначение |
|------|------------|
| `/etc/sudoers` | основной файл |
| `/etc/sudoers.d/*` | дополнительные файлы (рекомендуется) |
