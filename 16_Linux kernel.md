# Ядро Linux, модули, устройства, udev

## Вопросы и ответы

### Вопрос 1: Какие архитектуры ядер операционной системы существуют?

**Ответ:** существуют монолитное, микроядро, гибридное, экзоядро и модульное ядро.

#### Основные типы

| Архитектура | Описание | Примеры ОС |
|-------------|----------|------------|
| **Монолитное** | всё ядро в одном большом файле, все драйверы встроены | Linux (ранние версии), Unix |
| **Модульное** | монолитное ядро с поддержкой динамической загрузки модулей | **современный Linux**, Windows |
| **Микроядро** | минимальное ядро, сервисы в пользовательском пространстве | QNX, Minix, Mach |
| **Гибридное** | смесь монолитного и микроядра | Windows NT, macOS (XNU) |

#### Что такое модульное ядро

- **Монолитное ядро** с возможностью **динамической загрузки/выгрузки** модулей
- Модули можно добавлять без перезагрузки системы
- Пример: современный Linux

#### Преимущества модульного ядра

| Плюс | Почему |
|------|--------|
| Гибкость | можно загружать драйверы по мере необходимости |
| Экономия памяти | не загружается всё подряд |
| Удобство обновления | не нужно пересобирать всё ядро |

```bash
# Просмотр версии ядра
uname -r
# 5.14.0-284.el9.x86_64

# Просмотр загруженных модулей
lsmod
```

> Современный Linux — это **модульное монолитное ядро**. Это значит, что ядро одно, но драйверы можно подгружать динамически.

### Вопрос 2: Что такое модули ядра?

**Ответ:** модули ядра — это программы, которые помогают ядру работать с устройствами, подключёнными к компьютеру или шине. Это как бы контейнеры для драйверов.

#### Простыми словами

- Модуль = драйвер устройства
- Можно загрузить в ядро **без перезагрузки системы**
- Можно выгрузить, когда устройство не нужно

#### Примеры модулей

| Модуль | Для чего |
|--------|----------|
| `nvidia` | драйвер видеокарты NVIDIA |
| `e1000` | драйвер сетевой карты Intel |
| `usb_storage` | драйвер USB-флешек |
| `snd_hda_intel` | драйвер звуковой карты |

#### Где живут модули

```bash
# Директория с модулями для текущего ядра
ls /lib/modules/$(uname -r)/kernel/
```

> Модуль = драйвер. Если подключил новую USB-флешку — ядро загрузит модуль `usb_storage` автоматически через udev.

### Вопрос 3: Как посмотреть список загруженных модулей?

**Ответ:** командой `lsmod`.

#### Пример вывода

```bash
lsmod
```

```
Module                  Size  Used by
nvidia              12345678  42
e1000                 147456  0
usb_storage            73728  1
snd_hda_intel          45056  3
```

#### Что означают колонки

| Колонка | Что показывает |
|---------|----------------|
| **Module** | имя модуля |
| **Size** | размер модуля в байтах |
| **Used by** | сколько процессов/модулей используют этот модуль и список зависимостей |

#### Другие способы

```bash
# Информация о конкретном модуле
modinfo usb_storage

# Поиск модулей по имени
lsmod | grep nvidia
```

### Вопрос 4: Как узнать, есть ли определённый модуль в системе?

**Ответ:** есть несколько способов.

| Способ | Команда | Что показывает |
|--------|---------|----------------|
| **Проверить, загружен ли модуль** | `lsmod \| grep module_name` | загружен ли модуль в данный момент |
| **Проверить, есть ли модуль в системе** | `modinfo module_name` | существует ли модуль (независимо от загрузки) |
| **Найти модуль в файловой системе** | `find /lib/modules/$(uname -r) -name "*module_name*"` | физическое расположение модуля |

#### Примеры

```bash
# Проверить, загружен ли модуль nvidia
lsmod | grep nvidia

# Проверить, существует ли модуль usb_storage
modinfo usb_storage

# Найти все модули с именем "usb"
find /lib/modules/$(uname -r) -name "*usb*" | head -10
```

#### Разница между `lsmod | grep` и `modinfo`

| Команда | Что проверяет |
|---------|---------------|
| `lsmod \| grep` | загружен ли модуль **прямо сейчас** |
| `modinfo` | существует ли модуль **в системе** (может быть не загружен) |

> Сначала `modinfo` — проверяем, есть ли модуль. Потом `lsmod | grep` — проверяем, загружен ли он.

### Вопрос 5: Для чего нужна директория /dev?

**Ответ:** в директории `/dev` отображаются все устройства, подключённые к системе, в виде специальных файлов.

#### Что такое файлы устройств

- Это **не обычные файлы**, а интерфейсы для взаимодействия с устройствами
- Чтение/запись в такой файл = чтение/запись на устройство

#### Примеры файлов в /dev

| Файл | Что означает |
|------|--------------|
| `/dev/sda` | первый жёсткий диск (SATA/SCSI) |
| `/dev/sda1` | первый раздел на первом диске |
| `/dev/nvme0n1` | первый NVMe SSD |
| `/dev/tty1` | первая виртуальная консоль |
| `/dev/ttyUSB0` | USB-последовательный порт |
| `/dev/input/mouse0` | мышь |
| `/dev/random` | генератор случайных чисел |
| `/dev/null` | "чёрная дыра" (всё, что туда пишется, исчезает) |
| `/dev/zero` | генератор нулей |

#### Типы устройств

| Тип | Описание | Пример |
|-----|----------|--------|
| **Блочные (block)** | чтение/запись блоками | диски (`sda`, `nvme0n1`) |
| **Символьные (character)** | чтение/запись потоком | мышь, клавиатура, последовательные порты |

#### Просмотр устройств

```bash
# Список всех устройств
ls -la /dev/

# Только блочные устройства
ls -la /dev/sd*

# Только символьные
ls -la /dev/tty*
```

> Всё в Linux — это файл. Устройства — не исключение. `/dev/sda` — это файл, который представляет жёсткий диск.

### Вопрос 6: Как посмотреть список и информацию об устройствах?

**Ответ:** существует несколько команд для просмотра информации об устройствах.

#### Основные команды

| Команда | Что показывает |
|---------|----------------|
| `lspci` | устройства на шине PCI (видеокарта, сетевая карта, контроллеры) |
| `lsusb` | устройства на шине USB (флешки, мышь, клавиатура) |
| `lscpu` | информация о процессоре (модель, ядра, архитектура) |
| `lsblk` | блочные устройства (диски, разделы) |
| `lshw` | подробная информация о всей системе (железо) |
| `lshw -short` | краткий список устройств |

#### Примеры использования

```bash
# PCI устройства
lspci
lspci -v       # подробно
lspci -vv      # очень подробно

# USB устройства
lsusb
lsusb -v       # подробно

# Информация о процессоре
lscpu

# Блочные устройства (диски)
lsblk
lsblk -f       # с файловыми системами

# Вся информация о железе
sudo lshw
sudo lshw -short   # кратко
```

#### Дополнительные команды

| Команда | Что показывает |
|---------|----------------|
| `lsdev` | все устройства из /proc |
| `dmidecode` | информация о BIOS и железе (только root) |
| `cat /proc/cpuinfo` | подробная информация о CPU |
| `cat /proc/meminfo` | информация о памяти |

### Вопрос 7: Как загрузить и выгрузить модуль ядра?

**Ответ:** с помощью команд `modprobe` и `modprobe -r`.

#### Основные команды

| Действие | Команда | Пример |
|----------|---------|--------|
| **Загрузить модуль** | `sudo modprobe module_name` | `sudo modprobe usb_storage` |
| **Выгрузить модуль** | `sudo modprobe -r module_name` | `sudo modprobe -r usb_storage` |
| **Выгрузить модуль (альтернатива)** | `sudo rmmod module_name` | `sudo rmmod usb_storage` |

#### Разница между `insmod`/`rmmod` и `modprobe`

| Команда | Особенности |
|---------|-------------|
| `insmod` | загружает модуль, но **не загружает зависимости** |
| `rmmod` | выгружает модуль (если никто не использует) |
| `modprobe` | загружает модуль **со всеми зависимостями** |
| `modprobe -r` | выгружает модуль и его зависимости (если не используются) |

#### Примеры

```bash
# Загрузить модуль с зависимостями
sudo modprobe nvidia

# Выгрузить модуль
sudo modprobe -r nvidia

# Показать зависимости модуля
modprobe --show-depends nvidia

# Загрузить модуль без автоматической загрузки зависимостей (не рекомендуется)
sudo insmod /path/to/module.ko
```

#### Важные замечания

| Что нужно знать | Почему |
|----------------|--------|
| Модуль нельзя выгрузить, если он используется | сначала останови процессы, использующие модуль |
| `modprobe` ищет модули в `/lib/modules/$(uname -r)/` | стандартная директория |
| Некоторые модули нельзя выгрузить | встроены в ядро или используются системой |

### Вопрос 8: Как запретить загрузку модуля при включении ОС?

**Ответ:** добавить модуль в чёрный список (blacklist) в директории `/etc/modprobe.d/`.

#### Как это сделать

```bash
# 1. Создать файл с расширением .conf
sudo nano /etc/modprobe.d/blacklist.conf

# 2. Добавить строку с именем модуля
blacklist module_name

# 3. Сохранить и выйти
```

#### Пример

```bash
# Запретить загрузку модуля nvidia
sudo nano /etc/modprobe.d/blacklist-nvidia.conf

# Добавить:
blacklist nvidia

# Обновить initramfs (на некоторых системах)
sudo update-initramfs -u   # Ubuntu/Debian
sudo dracut -f             # CentOS/RHEL/Fedora
```

#### Другие способы

| Способ | Команда | Описание |
|--------|---------|----------|
| **Blacklist** | `blacklist module` | запрещает автоматическую загрузку |
| **Install** | `install module /bin/false` | запрещает загрузку любым способом |
| **Alias** | `alias module off` | отключает модуль |

#### Примеры в файлах

```bash
# /etc/modprobe.d/blacklist.conf
blacklist nvidia
blacklist nouveau
blacklist pcspkr

# /etc/modprobe.d/disable.conf
install nvidia /bin/false
install bluetooth /bin/false
```

#### Проверка

```bash
# Посмотреть все blacklist
grep -r "blacklist" /etc/modprobe.d/

# Проверить, загружен ли модуль после перезагрузки
lsmod | grep module_name
```

#### Важные замечания

| Нюанс | Почему |
|-------|--------|
| Blacklist не выгружает уже загруженный модуль | нужно перезагрузиться |
| Blacklist работает только для автоматической загрузки | можно загрузить вручную через `modprobe` |
| `install /bin/false` — более жёсткий запрет | даже ручная загрузка не сработает |

### Вопрос 9: Зачем нужен udev?

**Ответ:** udev — это программа, которая обрабатывает события при подключении/отключении устройств. Она даёт имена устройствам, загружает нужные модули и создаёт файлы в `/dev`.

#### Что делает udev

| Действие | Что происходит |
|----------|----------------|
| **Обнаружение устройства** | ядро сообщает udev о новом устройстве |
| **Загрузка модуля** | udev загружает нужный драйвер |
| **Создание файла в /dev** | устройство получает имя (например, `/dev/sdb1`) |
| **Применение правил** | выполняются действия из правил udev |

#### Где живут правила udev

```bash
# Пользовательские правила (создаются администратором)
/etc/udev/rules.d/

# Системные правила (не редактировать)
/lib/udev/rules.d/
```

#### Пример правила udev

```bash
# /etc/udev/rules.d/99-usb.rules
# При подключении USB-флешки выполнить скрипт
SUBSYSTEM=="block", ACTION=="add", RUN+="/usr/local/bin/usb-notify.sh"
```

#### Как работает udev

```
Подключение USB → ядро видит устройство → uevents → udev → 
→ загрузка модуля → создание /dev/sdb → применение правил
```

#### Полезные команды

```bash
# Мониторинг событий udev в реальном времени
sudo udevadm monitor

# Просмотр информации об устройстве
sudo udevadm info -a -n /dev/sda

# Применить правила udev вручную
sudo udevadm trigger

# Перезагрузить правила
sudo udevadm control --reload-rules
```

#### Зачем это нужно на стажировке

| Ситуация | Как помогает udev |
|----------|-------------------|
| Подключил флешку | udev создал `/dev/sdb1`, загрузил `usb_storage` |
| Подключил принтер | udev создал `/dev/usb/lp0` |
| Нужно автоматически монтировать флешку | написать правило udev |
| Устройство не появляется в `/dev` | проверить правила udev |

> udev — это менеджер устройств в Linux. Он отвечает за то, чтобы при подключении флешки появлялся файл `/dev/sdb1` и загружался нужный драйвер.

### Вопрос 10: Как посмотреть последние логи ядра?

**Ответ:** с помощью команд `dmesg` или `journalctl -k`.

#### Основные команды

| Команда | Что показывает |
|---------|----------------|
| `dmesg` | кольцевой буфер сообщений ядра |
| `journalctl -k` | сообщения ядра из systemd журнала |
| `dmesg -T` | логи ядра с человекочитаемым временем |
| `dmesg \| tail -20` | последние 20 строк |

#### Примеры использования

```bash
# Показать все логи ядра
dmesg

# Показать с нормальным временем
dmesg -T

# Последние 10 сообщений
dmesg | tail -10

# Логи ядра через journalctl
sudo journalctl -k

# Последние сообщения ядра через journalctl
sudo journalctl -k -n 20

# Логи ядра за последние 5 минут
sudo journalctl -k --since "5 minutes ago"
```

#### Когда что использовать

| Ситуация | Команда |
|----------|---------|
| Быстро посмотреть логи | `dmesg \| tail` |
| Нужно нормальное время | `dmesg -T` |
| Система с systemd | `journalctl -k` |
| Нужно фильтровать по времени | `journalctl -k --since` |

#### Типичные сообщения в логах ядра

| Сообщение | Что означает |
|-----------|--------------|
| `usb 1-1: new device` | подключилось USB-устройство |
| `sd 2:0:0:0: Attached` | обнаружен диск |
| `Out of memory` | не хватает памяти (OOM) |
| `segfault at ...` | ошибка сегментации |
| `CPU0: Temperature above threshold` | перегрев процессора |

#### Просмотр логов в реальном времени

```bash
# Следить за логами ядра в реальном времени
sudo dmesg -w

# Или через journalctl
sudo journalctl -k -f
```
