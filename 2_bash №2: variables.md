# bash №2: переменные

## Вопросы

### Вопрос 1: Чем отличаются локальные переменные от переменных окружения?

**Ответ:**

| | Локальные переменные | Переменные окружения |
|---|---------------------|----------------------|
| Назначение | для задач пользователя | для работы программ |
| Жизнь | только в текущей сессии | передаются в дочерние процессы |
| Экспорт | не требуют | требуют `export` |

**Проверка:**
```bash
# Локальная переменная
mylocal="hello"
env | grep mylocal   # (пусто) — не видна

# Переменная окружения
export myenv="world"
env | grep myenv     # myenv=world — видна
```

### Вопрос 2: Я задал переменную в ~/.bashrc (files=file1) и она не является переменной окружения. Но при этом она работала в дочерних bash сессиях. Почему?

**Ответ:**

Она работала не потому, что передалась по наследству, а потому что **каждая новая Bash-сессия заново читает файл `~/.bashrc`** и создаёт эту переменную заново.

```bash
# Эксперимент
# В .bashrc: files="file1"

# Запускаем дочерний Bash
bash
echo $files   # file1 (не унаследовано, а пересоздано!)
```

**Вывод:** Если переменная записана в `.bashrc` — она будет в каждой новой сессии, но это **не наследование**, а **пересоздание**.

## Практические задания

### Задание 1: Переменная curtime

**Мой способ (в `~/.bashrc`):**
```bash
curtime=date
```

**Как работает:**
```bash
sancher@centos8:~$ echo $curtime
date
sancher@centos8:~$ $curtime
Thu Apr 16 17:51:12 CEST 2026
sancher@centos8:~$ $curtime
Thu Apr 16 17:51:15 CEST 2026   # время обновляется!
```

**Почему обновляется?**  
В переменной хранится **строка `date`**, которая при вызове `$curtime` подставляется и выполняется как команда.

**Другие способы:**

| Способ | Команда в .bashrc | Результат |
|--------|-------------------|-----------|
| Строка с именем команды | `curtime=date` | `$curtime` выполняет `date` |
| Подстановка результата | `curtime=$(date)` | время замораживается |
| Алиас | `alias curtime='date'` | всегда свежее время |
| Функция | `curtime() { date; }` | всегда свежее время |

### Задание 2: Переменная user@hostname

**Мой способ:**
```bash
name="$LOGNAME@$HOSTNAME"
```

**Проверка:**
```bash
sancher@centos8:~$ echo $name
sancher@centos8
```

**Другие варианты:**

| Способ | Команда |
|--------|---------|
| Через `$USER` | `name="$USER@$HOSTNAME"` |
| Через `whoami` | `name="$(whoami)@$(hostname)"` |
| Алиас | `alias userhost='echo "$USER@$HOSTNAME"'` |
| Функция | `userhost() { echo "$USER@$HOSTNAME"; }` |

**Чтобы переменная была всегда:**
```bash
export name="$LOGNAME@$HOSTNAME"   # добавить в ~/.bashrc
```

## Файлы конфигурации Bash

| Файл | Когда загружается | Что добавлять |
|------|-------------------|---------------|
| `~/.bashrc` | интерактивная оболочка без логина (терминал) | алиасы, функции, PS1, локальные переменные |
| `~/.bash_profile` | интерактивная оболочка с логином (SSH, tty) | переменные окружения (`export`) |
| `~/.profile` | если нет `.bash_profile` | кросс-совместимость |


- **Алиасы и функции** → `~/.bashrc`
- **Переменные окружения** → `~/.bash_profile` или `~/.bashrc` с `export`

## Настройка русской локали (чтобы работал русский язык)


**Исправление:**
```bash
# Добавить строку
echo 'export LANG=ru_RU.UTF-8' >> ~/.bashrc

# Применить
source ~/.bashrc
```


### Варианты локалей

| Локаль | Сообщения команд | Русские буквы |
|--------|-----------------|---------------|
| `ru_RU.UTF-8` | на русском | отображаются |
| `en_US.UTF-8` | на английском | отображаются |
| `C` | на английском | непонятные символы |
