
# Вопросы и ответы по теме «Creating and copying files»

## 1. Что делает команда `touch`?

**Ответ:** обновляет время доступа к файлу (записывает в inode). Если файла не существует — создаёт его.

```bash
touch file.txt           # обновить время или создать
touch file{1..5}.txt     # создать 5 файлов одной командой
```

---

## 2. Чем скрытые файлы отличаются от обычных?

**Ответ:** скрытые файлы не видны при обычном просмотре командой `ls`. Они начинаются с точки (`.`). Чтобы их увидеть, нужно использовать `ls -a` (или `Ctrl+H` в графическом интерфейсе).

```bash
touch .hidden            # создаём скрытый файл
ls                       # не видно
ls -a                    # видно: .  ..  .hidden
```

---

## 3. Для чего нужен ключ `-v` (verbose)?

**Ответ:** ключ `-v` показывает действие команды в терминале — то есть что именно она сделала.

```bash
cp -v file.txt dir/      # 'file.txt' -> 'dir/file.txt'
rm -v old.txt            # removed 'old.txt'
mkdir -v newdir          # mkdir: created directory 'newdir'
```

---

## 4. Какой синтаксис у команды `cp`, если нужно скопировать несколько файлов и директорий?

**Ответ:** можно перечислить файлы по отдельности, указав в конце директорию назначения, либо использовать маски (wildcards) — например `file*`.

```bash
# По отдельности
cp file1.txt file2.txt file3.txt dir/

# Маской
cp *.txt dir/
cp file* dir/
```

> **Правило:** директория назначения всегда последняя.

---

## 5. Зачем нужен ключ `-i` у команды `cp`?

**Ответ:** ключ `-i` (interactive) нужен, чтобы мы сами решали — перезаписывать файл при копировании, если он уже существует, или нет.

```bash
cp -i file.txt dir/
# cp: overwrite 'dir/file.txt'? 
# y — перезаписать, n — пропустить
```

---

## 6. Как можно использовать несколько ключей одновременно?

**Ответ:** можно писать ключи слитно (например `-vi`) или по отдельности (`-v -i`). Порядок не имеет значения.

```bash
# Все эти команды делают одно и то же
cp -vi file.txt dir/
cp -iv file.txt dir/
cp -v -i file.txt dir/
```

---

## 7. Как можно использовать `*` и `{}` при работе с файлами?

**Ответ:**  
- `*` (звёздочка) — заменяет **любое количество любых символов** (означает "все")  
- `{}` (фигурные скобки) — генерируют **список вариантов** (например `{1..5}` от 1 до 5)

```bash
# * — для поиска существующих файлов
ls *.txt
ls file*

# {} — для генерации имён
touch file{1..5}.txt
touch {jpg,png,gif}/image.jpg
```

| Символ | Что делает | Пример |
|--------|------------|--------|
| `*` | Любое количество любых символов | `*.txt` |
| `?` | Один любой символ | `file?.txt` |
| `{1..5}` | Последовательность чисел | `file{1..5}` |
| `{a,b,c}` | Перечисление | `image.{jpg,png}` |

---

## Итог

| Команда/Ключ | Назначение |
|--------------|------------|
| `touch` | Обновить время / создать пустой файл |
| `.` (точка) | Скрытый файл или директория |
| `ls -a` | Показать скрытые файлы |
| `-v` | Подробный вывод (verbose) |
| `-i` | Спросить перед перезаписью |
| `*` | Маска "все" |
| `{}` | Генерация имён |
```

## Задание 1: Создание 5 файлов тремя способами

### (1) Пятью отдельными командами
```bash
touch file1
touch file2
touch file3
touch file4
touch file5
```

### (2) Одной командой (перечислением)
```bash
touch file11 file12 file13 file14 file15
```

### (3) Одной командой (фигурные скобки)
```bash
touch file{6..10}
```

**Результат:** созданы файлы `file1`...`file15`

## Задание 2: Создание скрытой директории и скрытого файла внутри

```bash
mkdir .secret_dir
cd .secret_dir
touch .secret_file
ls -a
# .  ..  .secret_file
```

* Скрытая директория `.secret_dir`  
* Скрытый файл `.secret_file` внутри

## Задание 3: Создание вложенных скрытых директорий одной командой

| Директория | Вложенность |
|------------|-------------|

| `.dirb`  | внутри `.dira` |
| `.dirc` | внутри `.dira` |

ls -a .dira/
# .  ..  .dirb  .dirc

## Задание 4: Создание скрытых копий файлов в скрытой директории

```bash
cp file1 ~/.dira/.file1
cp file2 ~/.dira/.file2
cp file3 ~/.dira/.file3
cp file4 ~/.dira/.file4
cp file5 ~/.dira/.file5
```

**Результат:** в `.dira` появились `.file1`...`.file5`

```bash
ls -la ~/.dira/
# .file1  .file2  .file3  .file4  .file5
```

## Задание 6: Создание жёсткой ссылки с помощью `cp -l`

```bash
cp -l file3 file6
```

**Проверка:**
```bash
ls -i
# 20443618 file1  20443620 file3  20443622 file5
# 20443619 file2  20443621 file4  20443620 file6
```

* Иноды `file3` и `file6` совпадают (`20443620`) — это жёсткая ссылка
