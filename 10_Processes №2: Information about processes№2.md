# Процессы №2: Информация о процессах №2

## Вопрос 1: Разбор вывода `top`

### Пример вывода
```
top - 21:49:10 up 6:48, 3 users, load average: 1.18, 0.89, 0.85
Tasks: 358 total, 4 running, 352 sleeping, 2 stopped, 0 zombie
%Cpu(s): 18.9 us, 4.4 sy, 0.0 ni, 76.7 id, 0.0 wa, 0.0 hi, 0.0 si, 0.0 st
MiB Mem : 3651.9 total, 340.7 free, 2754.2 used, 897.7 buff/cache
MiB Swap: 2048.0 total, 1432.7 free, 615.3 used, 897.7 avail Mem

  PID USER  PR  NI    VIRT    RES   SHR S  %CPU  %MEM   TIME+ COMMAND
 1234 user  20   0  4.2g  1.2g  200M S  18.9   8.5  00:55 firefox
```

### Строка 1: Статистика системы

| Поле | Что означает |
|------|--------------|
| `21:49:10` | текущее время |
| `up 6:48` | система работает 6 часов 48 минут |
| `3 users` | количество залогиненных пользователей |
| `load average: 1.18, 0.89, 0.85` | средняя нагрузка за 1, 5, 15 минут |

### Строка 2: Состояние процессов (Tasks)

| Поле | Что означает |
|------|--------------|
| `358 total` | всего процессов |
| `4 running` | выполняется прямо сейчас |
| `352 sleeping` | спят (ждут событий) |
| `2 stopped` | остановлены (SIGSTOP) |
| `0 zombie` | зомби-процессы (мёртвые, не похороненные) |

### Строка 3: Использование CPU

| Поле | Что означает |
|------|--------------|
| `us` (user) | время в пользовательском режиме |
| `sy` (system) | время в режиме ядра |
| `ni` (nice) | время процессов с изменённым приоритетом |
| `id` (idle) | процессор простаивает |
| `wa` (iowait) | ждёт ввода-вывода (диск) |
| `hi` (hardware interrupt) | аппаратные прерывания |
| `si` (software interrupt) | программные прерывания |
| `st` (steal) | время "украденное" гипервизором |

### Строка 4-5: Память

| Поле | Что означает |
|------|--------------|
| `total` | всего памяти |
| `free` | полностью свободна |
| `used` | реально используется |
| `buff/cache` | буферы и кэш |
| `avail Mem` | реально доступно для новых программ |

### Таблица процессов

| Столбец | Что означает |
|---------|--------------|
| `PID` | идентификатор процесса |
| `USER` | владелец процесса |
| `PR` | приоритет (чем меньше, тем выше) |
| `NI` | nice (вежливость) |
| `VIRT` | виртуальная память (вся) |
| `RES` | физическая память (в ОЗУ) |
| `SHR` | разделяемая память |
| `S` | состояние процесса |
| `%CPU` | процент использования CPU |
| `%MEM` | процент использования памяти |
| `TIME+` | процессорное время |
| `COMMAND` | команда |

## Вопрос 2: Зомби-процессы (zombie)

### Что это такое

**Зомби** — это процесс, который уже **завершился**, но всё ещё остаётся в таблице процессов.

### Как это происходит

1. Дочерний процесс завершает работу
2. Он отправляет сигнал **SIGCHLD** родительскому процессу
3. Родитель должен вызвать `wait()` чтобы "прочитать" статус завершения
4. Если родитель **не вызывает `wait()`** — дочерний процесс становится **зомби**

### Сравнение процессов

| Обычный процесс | Зомби-процесс |
|----------------|---------------|
| имеет память, ресурсы | не имеет ресурсов (все отдал) |
| может выполняться | не выполняется |
| занимает место в памяти | занимает **только запись в таблице процессов** |
| его можно убить (`kill -9`) | его **нельзя убить** (он уже мёртв) |

### Как убрать зомби

```bash
# Найти зомби-процессы (статус Z)
ps aux | grep Z

# Зомби можно убрать только убив родителя
kill -9 PPID   # где PPID — родитель зомби

# Или перезапустить родительский процесс
systemctl restart <service>
```

### Почему зомби — это плохо

- Таблица процессов имеет ограниченный размер
- Много зомби → не могут создаваться новые процессы
- Симптом: `fork: retry: Resource temporarily unavailable`

## Вопрос 3: Ограничение количества процессов

### Зачем это нужно?

1. **Защита от fork-бомб** — пользователь (или вирус) создаёт бесконечное количество процессов
2. **Стабильность системы** — один пользователь не может "положить" сервер
3. **Справедливое распределение ресурсов** — один пользователь не мешает другим

### Способы ограничения

#### Способ 1: `ulimit -u` (временный, для текущей сессии)

```bash
# Ограничить текущую сессию 100 процессами
ulimit -u 100

# Проверить
ulimit -u
```

#### Способ 2: `~/.bash_profile` (для конкретного пользователя)

```bash
# Добавить в ~/.bash_profile
ulimit -u 200
```

#### Способ 3: `/etc/security/limits.conf` (системный, для всех)

```bash
# /etc/security/limits.conf

# Для конкретного пользователя
sancher       soft    nproc       1024
sancher       hard    nproc       2048

# Для всех пользователей (*)
*             soft    nproc       1024
*             hard    nproc       2048

# Для root (лучше не ограничивать)
root          hard    nproc       unlimited
```

### Ограничение памяти

```bash
# /etc/security/limits.conf
sancher       soft    as         2097152   # 2GB виртуальной памяти
sancher       hard    as         4194304   # 4GB максимум

# Или через ulimit
ulimit -v 1048576   # 1GB
```

### Soft vs Hard лимиты

| Тип | Можно повысить | Можно понизить | Кто меняет |
|-----|---------------|----------------|------------|
| **soft** | да (до hard) | да | обычный пользователь |
| **hard** | нет (только root) | нет | только root |

### Проверка лимитов

```bash
# Текущие лимиты
ulimit -a

# Только количество процессов
ulimit -u

# Только виртуальную память
ulimit -v

# Лимиты конкретного процесса
cat /proc/PID/limits
```

### Что произойдёт при превышении лимита

```bash
# При попытке создать больше процессов:
bash: fork: retry: Resource temporarily unavailable

# При попытке выделить больше памяти:
Killed
```

### Как исправить, если переограничил себя

1. **Если ещё есть открытый терминал**:
   ```bash
   ulimit -u unlimited
   ```

2. **Если не могу зайти в систему**:
   - Загрузиться в rescue mode (single user)
   - Исправить `/etc/security/limits.conf`
   - Перезагрузиться

### Основные команды

| Команда | Что делает |
|---------|-------------|
| `top` | мониторинг в реальном времени |
| `htop` | улучшенная версия top |
| `ps -ef` | список всех процессов |
| `ps aux` | список всех процессов (другой формат) |
| `ps -u user` | процессы пользователя |


### Состояния процессов

| Буква | Состояние |
|-------|-----------|
| `R` | Running (выполняется) |
| `S` | Sleeping (спит) |
| `D` | Uninterruptible sleep (ждёт диск) |
| `Z` | Zombie (зомби) |
| `T` | Stopped (остановлен) |

### Лимиты

| Команда | Что ограничивает |
|---------|------------------|
| `ulimit -u` | количество процессов |
| `ulimit -v` | виртуальная память |
| `ulimit -m` | физическая память |
| `ulimit -n` | открытые файлы |
