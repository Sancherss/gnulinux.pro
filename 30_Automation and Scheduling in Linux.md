# Автоматизация и планировщики в Linux

## 1. Теоретическая часть (Вопросы и ответы)

### 1.1 Для чего нужен планировщик задач?
Планировщик нужен для автоматизации процессов в IT-инфраструктуре. Благодаря ему скрипты, бэкапы или системные отчеты выполняются в определенное время без участия человека, что исключает «человеческий фактор».

### 1.2 Чем отличаются планировщики задач друг от друга?
* **at**: Используется для разовых задач. Выполнил один раз в назначенное время и забыл.
* **cron**: Основной инструмент для цикличных, повторяющихся задач. Идеален для серверов 24/7.
* **anacron**: «Совестливый» планировщик для систем, которые часто выключаются (ноутбуки). Если время задачи было пропущено, пока ПК был выключен, anacron выполнит её сразу после загрузки.

### 1.3 В каких случаях стоит использовать cron, а в каких — anacron?
* **cron**: Когда сервер работает постоянно. Если время задачи пропущено (сервер лежал), cron не будет ее «догонять», а просто дождется следующего цикла.
* **anacron**: На домашних ПК. Минимальный интервал — 1 день. Гарантирует выполнение, даже если график работы устройства нестабилен.

### 1.4 Расшифровка записи: `0 4 8-14 * *`
* `0`: 0-я минута.
* `4`: 4 часа утра.
* `8-14`: С 8 по 14 число месяца (включительно).
* `*` (3-я): Любой месяц.
* `*` (4-я): Любой день недели.
**Итог:** Задача выполняется ежедневно в 04:00 в течение второй недели каждого месяца (с 8 по 14 число).

### 1.5 Использование планировщика на высоконагруженных серверах
Если время не критично, лучше использовать команду **batch**. Она ставит задачу в очередь и запускает её только тогда, когда средняя нагрузка системы (Load Average) падает ниже 0.8. Это предотвращает перегрузку сервера.

### 1.6 Как проверить задачи пользователей в системе?
1.  **Просмотр файлов:** `ls /var/spool/cron` (нужны права root).
2.  **Просмотр через команду:** `crontab -l` (свои задачи) или `sudo crontab -u username -l` (задачи конкретного пользователя).

## 2. Практическая часть (Задания)

### Задание №1: Скрипт с запуском раз в минуту
Был создан скрипт `/home/sancher/task1`, создающий файл. 

**Содержимое скрипта:**
```bash
#!/usr/bin/env bash
touch /data/file_$(date +%M) # создаем файл с минутой в названии
```

**Настройка в crontab (`crontab -e`):**
```bash
* * * * * /home/sancher/task1
```

### Задание №2: Разовая задача через `at`
Задача: создать файл через 3 минуты.
**Команды в терминале:**
```bash
sancher@centos8:~$ at now + 3 minutes
at> touch /data/at_file
at> <EOT> (нажато Ctrl+D)
job 5 at Sun Apr 26 10:55:00 2026
```

### Задание №3: Создание директории при перезагрузке
Для этого использовался специальный алиас `@reboot`.

**Запись в crontab:**
```bash
@reboot mkdir -p /data/boot_directory
```
