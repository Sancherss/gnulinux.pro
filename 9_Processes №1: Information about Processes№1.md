# Процессы №1: Информация о процессах №1

## Вопрос

### Запустите `ps -ef | head` и объясните значение каждого столбика.

**Команда:**
```bash
ps -ef | head
```

**Пример вывода:**
```
UID   PID  PPID   C STIME   TTY     TIME CMD
root    1     0   0 08:00   ?     00:00:01 /usr/lib/systemd/systemd
root    2     0   0 08:00   ?     00:00:00 [kthreadd]
root    3     2   0 08:00   ?     00:00:00 [rcu_gp]
```

### Значение столбцов

| Столбец | Что означает |
|---------|--------------|
| **UID** | пользователь, который запустил процесс |
| **PID** | уникальный идентификатор процесса |
| **PPID** | PID родительского процесса (кто запустил этот процесс) |
| **C** | загрузка процессора (использование ядра в процентах, округлённое) |
| **STIME** | время запуска процесса |
| **TTY** | терминал, к которому привязан процесс (`?` — нет терминала) |
| **TIME** | суммарное время, которое процесс провёл в режиме ядра и пользователя |
| **CMD** | команда, запустившая процесс |

### Уточнения

- **`C`** — мгновенная загрузка CPU в процентах
- **`TIME`** — не "сколько работал процесс", а **сколько процессорного времени он потребил**
- **`STIME`** — может показывать время или дату (если процесс старый)

## Практические задания

### Задание 1: Список процессов моего пользователя

**Команда:**
```bash
ps -u sancher
# или универсально:
ps -u $(whoami)
```

**Пример вывода:**
```
  PID TTY          TIME CMD
 1234 pts/0    00:00:01 bash
 4567 pts/0    00:00:00 ps
 7890 ?        00:00:02 firefox
```

**Что означают колонки:**

| Колонка | Значение |
|---------|----------|
| PID | идентификатор процесса |
| TTY | терминал (`?` — нет терминала) |
| TIME | процессорное время |
| CMD | команда |

**Другие полезные форматы:**
```bash
ps -u sancher -o pid,ppid,cmd   # только нужные колонки
ps -u sancher -f                 # полный формат
```

### Задание 2: Процессы в эмуляторах терминала

**Что такое эмулятор терминала?**  
Программы, которые создают псевдотерминалы (`pts/*`): `gnome-terminal`, `ptyxis`, `konsole`, `xterm`, `terminator`.

**Как найти все процессы в эмуляторах терминала:**

```bash
# Способ 1 (мой)
ps -ef | awk '$6 ~ /^pts\//'

# Способ 2
ps -e -o pid,tty,cmd | grep 'pts/'

# Способ 3 (для каждого pts отдельно)
ps -t pts/0
ps -t pts/1
```

**Как узнать, какие `pts` сейчас есть:**
```bash
who
# user   pts/0        08:00
# user   pts/1        09:30

ls /dev/pts/
# 0  1  2  ptmx
```

### Задание 3: Цепочка родительских процессов

**Задача:** запустить процесс, найти его PID, затем PPID и так до самого верха (до PID 1).

#### Мой эксперимент (вместо gedit использовал nano)

1. **Запустил процесс в фоне**
   ```bash
   nano file &
   # [1] 28114
   ```

2. **Нашёл цепочку родителей через `pstree -s`**
   ```bash
   pstree -s 28114
   ```
   **Результат:**
   ```
   systemd───systemd───ptyxis───ptyxis-agent───bash───bash───bash───bash───bash───bash───nano
   ```

3. **Проверил каждый шаг вручную через `ps -ef | grep PID`**

   | Шаг | PID | PPID | Команда |
   |-----|-----|------|---------|
   | 1 | 28114 | 27443 | `nano file` |
   | 2 | 27443 | 27162 | `bash` |
   | 3 | 27162 | 26918 | `bash` |
   | 4 | 26918 | 23354 | `bash` |
   | 5 | 23354 | 15448 | `bash` |
   | 6 | 15448 | 14465 | `bash` |
   | 7 | 14465 | 3761 | `/usr/bin/bash` |
   | 8 | 3761 | 3747 | `/usr/libexec/ptyxis-agent` |
   | 9 | 3747 | 2734 | `/usr/bin/ptyxis` |
   | 10 | 2734 | 1 | `/usr/lib/systemd/systemd --user` |
   | 11 | 1 | 0 | `/usr/lib/systemd/systemd --switched-root ...` |

#### Итоговая цепочка

```
nano (28114)
  ↑ PPID=27443
bash (27443)
  ↑ PPID=27162
bash (27162)
  ↑ PPID=26918
bash (26918)
  ↑ PPID=23354
bash (23354)
  ↑ PPID=15448
bash (15448)
  ↑ PPID=14465
/usr/bin/bash (14465)
  ↑ PPID=3761
/usr/libexec/ptyxis-agent (3761)
  ↑ PPID=3747
/usr/bin/ptyxis (3747)
  ↑ PPID=2734
/usr/lib/systemd/systemd --user (2734)
  ↑ PPID=1
/usr/lib/systemd/systemd --switched-root ... (1)
  ↑ PPID=0
```

#### Важные наблюдения

- **PID 1** — это `systemd` (главный процесс системы, предок всех процессов)
- **PPID 0** — у процесса с PID 1 (такого процесса не существует, это просто маркер)
- Процессы с TTY `?` — системные процессы/демоны, не привязанные к терминалу
- `pts/*` — псевдотерминалы (эмуляторы терминала)

### Основные команды

| Команда | Что делает |
|---------|-------------|
| `ps -ef` | все процессы (полный формат) |
| `ps aux` | все процессы (другой формат) |
| `ps -u user` | процессы пользователя |
| `ps -t pts/0` | процессы на конкретном терминале |
| `pstree -p PID` | дерево процессов |
| `pstree -s PID` | цепочка родителей |
| `pgrep name` | найти PID по имени |

### Поля для `ps -o`

```bash
ps -o pid,ppid,uid,user,cmd,%cpu,%mem,stat,tty,time
```

| Поле | Что означает |
|------|--------------|
| `pid` | идентификатор процесса |
| `ppid` | родительский PID |
| `uid` | ID пользователя |
| `user` | имя пользователя |
| `cmd` | команда |
| `%cpu` | использование CPU |
| `%mem` | использование памяти |
| `stat` | состояние процесса |
| `tty` | терминал |
| `time` | процессорное время |
