
# Systemd

## 1. Теоретические вопросы

### Что такое система инициализации?
Это самый первый процесс, запускаемый ядром после загрузки (**PID 1**). Она работает в пространстве пользователя и отвечает за запуск всех остальных программ и служб, необходимых для полноценной работы ОС. Если этот процесс завершится, система уйдет в критическую ошибку (*kernel panic*).

### Какая система инициализации используется в нашем дистрибутиве?
В современных дистрибутивах (Fedora, AlmaLinux, CentOS 8+, Debian, Ubuntu) используется **systemd**.

### Что такое таргеты (targets)?
Это специальные юниты systemd, которые группируют другие юниты (сервисы, сокеты, другие таргеты). Они определяют состояние системы и являются гибкой заменой старым уровням выполнения (*runlevels*).

### Какой таргет используется по умолчанию и как это проверить?
* В Fedora (с GUI) обычно используется: `graphical.target`.
* В серверных ОС (AlmaLinux без GUI) обычно используется: `multi-user.target`.
* **Команда для проверки:**
  ```bash
  systemctl get-default
  ```

### Как посмотреть, какие сервисы входят в таргет?
Для этого используется дерево зависимостей:
```bash
systemctl list-dependencies <имя_таргета>
```

### Что такое демоны?
Это процессы, которые работают в фоновом режиме, не имеют управляющего терминала и обычно ожидают каких-либо событий или запросов (например, `sshd` ожидает подключения по сети).

### Что такое сервисы?
Это юниты-инструкции для systemd. Сервисный файл описывает, как именно управлять демоном: путь к исполняемому файлу, условия запуска, зависимости и правила перезапуска.

### Как узнать команду запуска сервиса?
Нужно посмотреть содержимое юнита (директива `ExecStart`):
```bash
systemctl cat <имя_сервиса>
```

### Управление таргетами
* **Поменять текущий (на лету):**
  ```bash
  sudo systemctl isolate <имя_таргета>
  ```
* **Поменять таргет по умолчанию (навсегда):**
  ```bash
  sudo systemctl set-default <имя_таргета>
  ```

### Управление автозагрузкой
* **Добавить в автозагрузку:** `sudo systemctl enable <сервис>`
* **Убрать из автозагрузки:** `sudo systemctl disable <сервис>`
* **Проверить, включен ли сервис:** `systemctl is-enabled <сервис>`

## 2. Практические задания

### Задание №1: Работа в rescue.target
**Цель:** Перейти в режим восстановления, изучить `fstab` и вернуться обратно.

1. **Переход в режим восстановления:**
   ```bash
   sudo systemctl isolate rescue.target
   ```
   *Система останавливает большинство служб и переходит в однопользовательский консольный режим.*

2. **Просмотр таблицы монтирования:**
   ```bash
   cat /etc/fstab
   ```

3. **Возврат в графический режим:**
   ```bash
   sudo systemctl isolate graphical.target
   ```

### Задание №2: Изменение режима по умолчанию
**Цель:** Установить консольный многопользовательский режим как основной при загрузке.

**Выполнение:**
```bash
sudo systemctl set-default multi-user.target
```

**Проверка результата:**
```bash
systemctl get-default
# Ожидаемый вывод: multi-user.target
```
