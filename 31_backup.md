# Создание backup скрипта
## Раздел 1. Ответы на вопросы

**1. Как посмотреть размер файлов и директорий?**
* Для директорий используется команда `du` (disk usage), обычно с ключами `-h` (human-readable) и `--max-depth=1` (чтобы видеть только первый уровень вложенности).
* Для отдельных файлов достаточно `ls -lh`.

**2. Чем отличается `;` от `+` у утилиты `find`?**
* При использовании **`\;`** команда после `-exec` запускается отдельно для **каждого** найденного файла.
* При использовании **`+`** `find` формирует из найденных файлов один длинный список и запускает команду **один раз** для всех сразу (что экономит ресурсы системы).

**3. Для чего нужна архивация? Какая утилита используется?**
* Архивация нужна для объединения множества файлов и структуры папок в один файл-контейнер (удобно для бэкапов или передачи данных).
* Основная утилита в Linux — **`tar`**.

**4. Для чего нужно сжатие? Какие утилиты используются, и чем они отличаются?**
* Сжатие нужно для уменьшения физического размера данных на диске.
* **`gzip`**: работает быстро, сжатие среднее.
* **`bzip2`**: работает медленнее, но сжимает текстовые данные (логи) сильнее, чем gzip.
* **`xz`**: самое сильное сжатие, но требует много времени и ресурсов процессора.

## Раздел 2. Выполнение заданий

### 1. Поиск самой большой директории (цепочка)
```bash
# В корне
sudo du -sh /* 2>/dev/null | sort -rh | head -1
# Внутри найденной (например, /usr)
sudo du -sh /usr/* 2>/dev/null | sort -rh | head -1
```

### 2. Поиск файлов текущего пользователя
```bash
find / -user sancher 2> /dev/null
```

### 3. Поиск директорий с «.d» в имени
```bash
find / -type d -name "*.d" > /home/sancher/directories.txt 2> /dev/null
```

### 4. Поиск SUID-файлов и копирование
```bash
mkdir ~/suidfiles
sudo find / -type f -perm /4000 -exec cp -v {} /home/sancher/suidfiles/ \;
```

### 5. Сравнение архивации gzip и bzip
```bash
# Создание архивов
sudo tar -czf /tmp/logs.tar.gz /var/log
sudo tar -cjf /tmp/logs.tar.bz2 /var/log
# Сравнение размеров
ls -lh /tmp/logs.tar.*
```

### 6. Работа с инодами и ссылками
```bash
mkdir exam && cd exam
touch myfile
ln myfile hardlink
ln -s myfile softlink
# Узнаем иноды (например, 292330 и 292331)
ls -i
mv myfile 292330
mv softlink 292331
# Создание архива и удаление из него символической ссылки
cd ..
tar -cf exam.tar exam/
tar --delete -f exam.tar exam/292331
gzip exam.tar
```

### 7. Настройка планировщика Cron
**Команды для `crontab -e` соответствующих пользователей:**

* **Для пользователя `backup`:**
    `0 0 * * * find /data -type f -mtime +5 -exec tar -rf /backup/archive.tar {} \; -delete`

* **Для пользователя `user`:**
    `*/10 * * * * touch /data/file_$(date +\%s)`

* **Для пользователя `root`:**
    `30 23 * * 5 { ls -l /data; tar -tf /backup/archive.tar; } >> /var/log/reports/$(date +\%F).log`
