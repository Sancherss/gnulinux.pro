## Вопросы и ответы

### Вопрос 1: Процесс запуска операционной системы

**Ответ:** процесс загрузки Linux проходит несколько этапов:

| Этап | Что происходит |
|------|----------------|
| **1. BIOS/UEFI** | самотестирование, выбор загрузочного устройства |
| **2. Загрузчик (GRUB)** | находит и загружает ядро и initramfs |
| **3. Ядро** | загружает встроенные модули |
| **4. initramfs** | временная ФС для дозагрузки модулей (диски, RAID, LVM) |
| **5. Корневая ФС** | монтируется в `/` |
| **6. init/systemd** | первый процесс, запуск остальных служб |

**Схема загрузки:**
```
BIOS/UEFI → GRUB → ядро → initramfs → корневая ФС → systemd (PID 1)
```

---

### Вопрос 2: Для чего нужен загрузчик?

**Ответ:** загрузчик нужен, чтобы загрузить ядро, чтобы оно уже начало запуск операционной системы.

**Основные функции загрузчика:**

| Функция | Описание |
|---------|----------|
| Загрузка ядра | находит образ ядра на диске и загружает его в память |
| Загрузка initramfs | загружает временную файловую систему с модулями |
| Передача параметров | передаёт ядру параметры (корневая ФС, режимы загрузки) |
| Выбор ОС | позволяет выбрать систему при мультизагрузке |

---

### Вопрос 3: Когда нужно выносить /boot в отдельный раздел?

**Ответ:** /boot нужно выносить в отдельный раздел в следующих случаях:

| Случай | Почему |
|--------|--------|
| GRUB Legacy | старый загрузчик не умеет читать LVM и некоторые ФС |
| Использование LVM | GRUB может не видеть LVM-тома |
| Шифрование диска | /boot должен быть не зашифрован, чтобы GRUB мог прочитать ядро |
| RAID | некоторые конфигурации RAID требуют отдельного /boot |
| Старые BIOS | ограничения на размер диска |

---

### Вопрос 4: Как изменить настройки загрузчика?

**Ответ:** для изменения настроек загрузчика GRUB нужно:

| Шаг | Действие | Команда |
|-----|----------|---------|
| 1 | Редактируем файл настроек | `sudo nano /etc/default/grub` |
| 2 | Генерируем новый конфиг | `sudo grub2-mkconfig -o /boot/grub2/grub.cfg` |

**Основные параметры в /etc/default/grub:**

| Параметр | Что делает |
|----------|------------|
| `GRUB_TIMEOUT=5` | время отображения меню |
| `GRUB_DEFAULT=0` | пункт меню по умолчанию |
| `GRUB_CMDLINE_LINUX` | параметры, передаваемые ядру |
| `GRUB_BACKGROUND` | путь к фоновому изображению |

---

### Вопрос 5: Для чего нужен initramfs?

**Ответ:** initramfs нужен для того, чтобы ядро могло примонтировать временную файловую систему для дозагрузки нужных модулей, чтобы примонтировать основную файловую систему.

**Как работает initramfs:**

| Этап | Что происходит |
|------|----------------|
| 1 | GRUB загружает ядро и initramfs в память |
| 2 | Ядро распаковывает initramfs как временную корневую ФС |
| 3 | Ядро запускает скрипты внутри initramfs |
| 4 | initramfs загружает недостающие модули (диски, RAID, LVM) |
| 5 | initramfs монтирует настоящую корневую ФС в `/` |
| 6 | Ядро переключается на реальную корневую ФС и запускает systemd |

---

### Вопрос 6: Чем отличается initrd от initramfs?

**Ответ:** initrd — старый формат (блочное устройство), initramfs — новый (cpio архив).

| Характеристика | initrd | initramfs |
|----------------|--------|-----------|
| Формат | блочное устройство | cpio архив |
| Способ работы | монтируется как блочное устройство | распаковывается в tmpfs |
| Размер | фиксированный, больше | динамический, компактнее |
| Скорость | медленнее | быстрее |
| Использование | старые ядра (до 2.6) | современные ядра (2.6+) |

---

### Вопрос 7: Как узнать, какие модули есть в initramfs?

**Ответ:** с помощью команды `lsinitrd`

| Команда | Что показывает |
|---------|----------------|
| `lsinitrd` | список модулей и файлов в initramfs текущего ядра |
| `lsinitrd /boot/initramfs-$(uname -r).img` | то же самое явно |
| `lsinitrd \| grep ".ko"` | только модули ядра |

```bash
# Пример
lsinitrd | grep virtio
```

---

### Вопрос 8: Как узнать, какая корневая ФС указана в GRUB?

**Ответ:** с помощью просмотра конфигурации GRUB и поиска параметра `root=`.

| Способ | Команда |
|--------|---------|
| grep в grub.cfg | `sudo grep "root=" /boot/grub2/grub.cfg` |
| grep linux | `sudo grep "linux" /boot/grub2/grub.cfg` |
| При загрузке | нажать `e` для редактирования |

```bash
# Пример вывода
linux /vmlinuz-5.14.0 root=/dev/mapper/almalinux_centos8-root ro quiet
```

## Практические задания

### Задание 1: Поменять фон загрузчика

**Выполнение:**

```bash
# 1. Скопировать изображение в директорию GRUB
sudo cp /path/to/image.png /boot/grub2/

# 2. Добавить в /etc/default/grub
echo 'GRUB_BACKGROUND="/boot/grub2/image.png"' | sudo tee -a /etc/default/grub

# 3. Сгенерировать новый конфиг
sudo grub2-mkconfig -o /boot/grub2/grub.cfg

# 4. Перезагрузиться
sudo reboot
```

**Примечание:** В VirtualBox может не отображаться меню GRUB. Чтобы его увидеть, нужно:
- При загрузке нажать `Esc` или `Shift`
- Или установить `GRUB_TIMEOUT=10` в `/etc/default/grub`

### Задание 2: Создать initramfs с модулями virtio

**Выполнение:**

```bash
# 1. Создать файл конфигурации
sudo nano /etc/dracut.conf.d/virtio.conf

# 2. Добавить модули
add_drivers+=" virtio virtio_blk virtio_net virtio_pci virtio_ring "

# 3. Пересобрать initramfs
sudo dracut -f -v /boot/initramfs-$(uname -r).img $(uname -r)

# 4. Проверить
lsinitrd | grep virtio
```

**Параметры dracut:**

| Параметр | Значение |
|----------|----------|
| `-f` | принудительно перезаписать |
| `-v` | подробный вывод |

### GRUB

| Команда | Что делает |
|---------|------------|
| `sudo nano /etc/default/grub` | редактирование настроек GRUB |
| `sudo grub2-mkconfig -o /boot/grub2/grub.cfg` | генерация конфига |
| `sudo grep "root=" /boot/grub2/grub.cfg` | поиск корневой ФС |

### initramfs

| Команда | Что делает |
|---------|------------|
| `lsinitrd` | просмотр содержимого initramfs |
| `lsinitrd \| grep ".ko"` | поиск модулей |
| `sudo dracut -f -v` | пересобрать initramfs |
| `sudo dracut --add-drivers "virtio"` | добавить модули |

### Директории и файлы

| Путь | Назначение |
|------|------------|
| `/etc/default/grub` | настройки GRUB |
| `/boot/grub2/grub.cfg` | конфиг GRUB |
| `/boot/initramfs-$(uname -r).img` | образ initramfs |
| `/etc/dracut.conf.d/` | конфигурация dracut |
| `/boot/` | директория с ядром и initramfs |
