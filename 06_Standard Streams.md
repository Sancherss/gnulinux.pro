# Стандартные потоки (stdin, stdout, stderr)

## Вопросы

### Вопрос 1: Перечислите стандартные потоки.

**Ответ:** стандартные потоки — это:

| Поток | Название | Описание | Номер (дескриптор) |
|-------|----------|----------|-------------------|
| `stdin` | стандартный ввод | данные, которые программа получает (например, с клавиатуры) | 0 |
| `stdout` | стандартный вывод | результат работы программы (обычно на экран) | 1 |
| `stderr` | стандартный вывод ошибок | сообщения об ошибках (отдельно от stdout) | 2 |

**Простыми словами:**
- **stdin (0)** — то, что ты вводишь с клавиатуры
- **stdout (1)** — то, что программа выводит на экран (результат)
- **stderr (2)** — то, что программа выводит на экран (ошибки)

---

### Вопрос 2: Как и куда можно направлять стандартные потоки?

**Ответ:** потоки можно перенаправлять в файлы с помощью специальных операторов.

#### Операторы перенаправления

| Оператор | Что делает | Пример |
|----------|------------|--------|
| `<` | берёт данные из файла (ввод) | `cat < file.txt` |
| `>` | перенаправляет **stdout** в файл (перезапись) | `ls > list.txt` |
| `>>` | перенаправляет **stdout** в файл (добавление) | `echo "hello" >> file.txt` |
| `2>` | перенаправляет **stderr** в файл (перезапись) | `ls /fake 2> error.log` |
| `2>>` | перенаправляет **stderr** в файл (добавление) | `command 2>> error.log` |
| `&>` | перенаправляет **stdout и stderr** в один файл | `command &> all.log` |

#### Примеры

```bash
# Ввод из файла
sort < names.txt

# Вывод в файл (перезапись)
echo "Новый текст" > file.txt

# Добавление в конец файла
echo "Дополнительная строка" >> file.txt

# Только ошибки в файл
ls /nonexistent 2> error.log

# И вывод, и ошибки в один файл
ls /home /nonexistent &> result.log

# Классический способ: stdout в один файл, stderr в другой
ls /home /nonexistent > out.log 2> err.log
```

> `2>&1` — означает "перенаправить stderr (2) туда же, куда и stdout (1)".

---

### Вопрос 3: Как направить стандартный вывод в терминал и в файл одновременно?

**Ответ:** с помощью команды `tee`.

```bash
# Основной синтаксис
команда | tee файл

# Пример
ls -la | tee list.txt
```

#### Полезные варианты

```bash
# Добавить в конец файла (не перезаписывать)
ls -la | tee -a list.txt

# Одновременно в несколько файлов
ls -la | tee file1.txt file2.txt

# Перенаправить и stdout, и stderr через tee
ls /home /nonexistent 2>&1 | tee output.log
```
---

### Вопрос 4: Как направить stdout и stderr в один файл и в разные файлы?

**Ответ:**

#### В один файл

```bash
# Способ 1 (современный)
команда &> файл

# Способ 2 (классический)
команда > файл 2>&1
```

#### В разные файлы

```bash
команда > stdout.log 2> stderr.log
```

#### Сводная таблица

| Задача | Команда |
|--------|---------|
| stdout в файл | `command > out` |
| stderr в файл | `command 2> err` |
| stdout и stderr в ОДИН файл | `command &> all` |
| stdout и stderr в РАЗНЫЕ файлы | `command > out 2> err` |
| stdout в файл, stderr на экран | `command > out` |
| stdout на экран, stderr в файл | `command 2> err` |

### Задание 2: Добавить строки 13–21 в файл

```bash
head -n 21 /etc/group | tail -n 8 >> mykernel
```

#### Разбор команды

| Часть | Что делает |
|-------|-------------|
| `head -n 21 /etc/group` | берёт первые 21 строку из `/etc/group` |
| `\|` | передаёт результат дальше (pipe) |
| `tail -n 8` | из 21 строки оставляет последние 8 (строки 14–21) |
| `>> mykernel` | добавляет в конец файла |

#### Альтернативные способы

```bash
# Через sed (самый прямой)
sed -n '13,21p' /etc/group >> mykernel
```

### Задание 4: Комбинированное перенаправление

**Мой вариант (через head/tail):**
```bash
head -n 20 /etc/group | tail -n +13 > ~/lines
head -n 16 /etc/passwd | tail -n 1 >> ~/lines
```

**Через sed (проще и быстрее):**
```bash
sed -n '13,20p' /etc/group > ~/lines
sed -n '16p' /etc/passwd >> ~/lines
```

### Перенаправление потоков

| Оператор | Назначение |
|----------|------------|
| `>` | stdout → файл (перезапись) |
| `>>` | stdout → файл (добавление) |
| `2>` | stderr → файл (перезапись) |
| `2>>` | stderr → файл (добавление) |
| `&>` | stdout + stderr → файл |
| `2>&1` | stderr → туда же, куда stdout |
| `\|` | передача данных между командами |

### Полезные команды

| Команда | Что делает |
|---------|-------------|
| `tee файл` | вывод на экран и в файл |
| `tail -f` | следить за логом в реальном времени |
| `sed -n 'm,np'` | вывести строки с m по n |
| `head -n +N` | начиная с N-й строки |

### Подавление вывода

```bash
# Не показывать ошибки
command 2> /dev/null

# Не показывать ничего
command &> /dev/null
```
