# Пакетный менеджер DNF

---

## 1. Что такое пакет и зачем он нужен?

- Программа в Linux — это не один файл, а набор компонентов:
    - исполняемые файлы (`/usr/bin`, `/usr/sbin`),
    - конфигурационные файлы (`/etc`),
    - библиотеки (определяются через `ldd`),
    - документация (`/usr/share/doc`),
    - служебные пользователи/сервисы.
- **Пакет** — специальный архив, который содержит:
    - файлы программы (распределённые по структуре корневой ФС),
    - метаинформацию (название, версия, зависимости, лицензия),
    - скрипты, выполняемые до/после установки/удаления (`%pre`, `%post`, `%preun`, `%postun`).

Пример распаковки RPM-пакета вручную:
```bash
rpm2cpio lvm2-2.03.32-3.el10.x86_64.rpm | cpio -idmv
```
После распаковки появляются директории `/etc`, `/run`, `/usr` — именно так пакет "ложится" в систему.

## 2. Форматы пакетов и пакетные менеджеры

| Дистрибутив | Формат | Низкоуровневый | Высокоуровневый (с зависимостями) |
|-------------|--------|----------------|----------------------------------|
| RHEL / CentOS / AlmaLinux | `.rpm` | `rpm` | `dnf` (или `yum` — симлинк) |
| Debian / Ubuntu | `.deb` | `dpkg` | `apt`, `aptitude` |

- **`rpm`** — низкоуровневая утилита:
    - умеет устанавливать (`-i`), удалять, запрашивать информацию (`-q`), но **не решает зависимости**.
    - хранит базу установленных пакетов в `/var/lib/rpm`.
- **`dnf`** — высокоуровневый менеджер:
    - автоматически разрешает зависимости,
    - работает с репозиториями,
    - умеет искать, обновлять, удалять пакеты.

## 3. Зависимости и репозитории

- **Зависимости** — необходимость в других пакетах (библиотеках, утилитах).
- **Репозиторий** — сервер с набором пакетов, поддерживаемый мейнтейнерами дистрибутива.
- Почему не нужно ставить всё вручную? Потому что `dnf` сам найдёт и установит нужные зависимости.
- Посмотреть зависимости пакета:
  ```bash
  dnf deplist lvm2
  ```
- Список подключённых репозиториев:
  ```bash
  dnf repolist
  dnf repolist -v   # подробнее
  ```

## 4. Информация о пакетах (rpm)

### Через `rpm` для установленного пакета:
```bash
rpm -ql lvm2          # список файлов пакета
rpm -qa               # все установленные пакеты
rpm -qlp lvm2*.rpm    # список файлов в ещё не установленном .rpm
rpm -qip lvm2*.rpm    # подробная информация о пакете
rpm -qp --scripts lvm2*.rpm  # скрипты (postinstall, preuninstall и т.д.)
```

### Пример содержимого метаинформации:
- Name, Version, Release, Architecture
- Epoch (для нестандартной версионности)
- Size, License, Build Date, Packager, URL, Summary, Description

## 5. Поиск и работа с пакетами в DNF

| Действие | Команда |
|----------|---------|
| Поиск пакета | `dnf search virtualbox` |
| Информация о пакете | `dnf info httpd` |
| Установка из репозитория | `sudo dnf install lvm2` |
| Установка локального `.rpm` | `sudo dnf install ./package.rpm` |
| Установка по URL | `sudo dnf install http://.../package.rpm` |
| Удаление | `sudo dnf remove VirtualBox-6.1` |
| Обновление всех пакетов | `sudo dnf upgrade` |
| Проверка обновлений | `dnf check-upgrade` |
| Нужна ли перезагрузка? | `sudo dnf needs-restarting -r` |
| Какой пакет даёт файл? | `dnf provides /usr/bin/nslookup` |
| История операций | `dnf history`, `dnf history info 32`, `sudo dnf history undo 32` |

## 6. Репозитории: подключение, ключи, проверка подписи

- Файлы репозиториев лежат в `/etc/yum.repos.d/*.repo`
- Структура `.repo`-файла:
  ```ini
  [vbox]
  name=VirtualBox repo
  baseurl=http://download.virtualbox.org/virtualbox/rpm/el8/$basearch
  enabled=1
  gpgcheck=1
  gpgkey=https://www.virtualbox.org/download/oracle_vbox.asc
  ```
- **gpgcheck** — проверка цифровой подписи пакетов (защита от MITM-атак).
- Добавление ключа вручную:
  ```bash
  wget https://example.com/key.asc
  sudo rpm --import key.asc
  ```

## 7. Группы и модули

- **Группы** — набор пакетов, устанавливаемых вместе (например, "Container Management"):
  ```bash
  dnf grouplist
  dnf groupinfo "Container Management"
  sudo dnf groupinstall "Container Management"
  ```
- **Модули** — группы пакетов с фиксированными версиями (потоки):
  ```bash
  dnf module list
  dnf module info container-tools:2.0
  sudo dnf module install container-tools:2.0
  ```

---

## 8. Кеш репозиториев и автоматическое обновление метаданных

- Метаданные репозиториев хранятся локально, обновляются через таймер:
  ```bash
  systemctl status dnf-makecache.timer
  ```
- Принудительное обновление кеша:
  ```bash
  dnf makecache
  ```

---

## 9. Важные замечания

- Не рекомендуется подключать репозитории от других дистрибутивов или сильно отличающихся версий — это ломает зависимости.
- `yum` в CentOS 8 — это симлинк на `dnf`.
- При установке из сторонних `.rpm`-файлов зависимости не всегда решаются автоматически (лучше использовать `dnf install ./file.rpm`).
- Проверка скриптов пакета перед установкой — хорошая практика безопасности:
  ```bash
  rpm -qp --scripts suspicious-package.rpm
  ```

---

## 10. Итог

- `dnf` — мощный инструмент управления пакетами в RHEL-семействе.
- Основные возможности:
    - установка/удаление/обновление ПО,
    - разрешение зависимостей,
    - работа с репозиториями,
    - поиск пакетов по файлам,
    - группы и модули,
    - история операций с откатом.
- `rpm` — низкоуровневая утилита для работы с `.rpm`-файлами (просмотр информации, извлечение, импорт ключей).
