# Вопросы и ответы: Просмотр содержимого файлов и поиск
---

## 1. С помощью каких команд можно посмотреть содержимое текстового файла?

**Ответ:** `cat`, `less`, `head`, `tail`

| Команда | Назначение | Пример |
|---------|------------|--------|
| `cat` | Выводит **весь** файл целиком | `cat file.txt` |
| `less` | Постраничный просмотр (можно листать) | `less file.txt` |
| `head` | Показывает **первые** 10 строк | `head file.txt` |
| `tail` | Показывает **последние** 10 строк | `tail file.txt` |

```bash
# Примеры
cat /etc/passwd
less /var/log/syslog          # q — выход
head -n 20 /etc/passwd
tail -n 5 /var/log/syslog
```

---

## 2. Какими способами можно посмотреть 30-ю строчку файла `/etc/passwd`?

**Ответ:** комбинацией `head` и `tail` или с помощью `sed`/`awk`

### Способ 1: `head` + `tail`
```bash
head -n 30 /etc/passwd | tail -n 1
```

### Способ 2: `sed`
```bash
sed -n '30p' /etc/passwd
```

### Способ 3: `awk`
```bash
awk 'NR==30' /etc/passwd
```

### Способ 4: `head` + `tail` с `+`
```bash
head -n 30 /etc/passwd | tail -n +30
```

---

## 3. Какое преимущество дают `head` и `tail` при чтении больших файлов?

**Ответ:** они **не читают файл целиком**. Мы можем прочитать либо первые `n` строк, либо последние, не загружая весь файл в память.

| Команда | Что делает | Память |
|---------|------------|--------|
| `cat huge.log` | Читает **весь** файл (10 ГБ) | Заполнит память |
| `head huge.log` | Читает **только первые 10 строк** | Мгновенно |
| `tail huge.log` | Читает **только последние 10 строк** | Мгновенно |

```bash
# Плохо для больших файлов
cat /var/log/huge.log

# Хорошо для больших файлов
head -n 50 /var/log/huge.log
tail -n 50 /var/log/huge.log
```

## 4. Как найти файл, в котором есть слово «hello»?

**Ответ:** использовать `grep -r` (рекурсивный поиск).

```bash
grep -r "hello" /путь/к/поиску
```

### Примеры
```bash
# Искать во всей системе
grep -r "hello" /

# Искать в текущей директории
grep -r "hello" .

# Показывать только имена файлов
grep -rl "hello" /

# Игнорировать регистр
grep -ri "hello" /
```

| Ключ | Назначение |
|------|------------|
| `-r` | рекурсивно (обходить подпапки) |
| `-l` | показывать только имена файлов |
| `-i` | игнорировать регистр |
| `-n` | показывать номера строк |

---

## 5. Каким удобным способом можно смотреть изменяющийся файл?

**Ответ:** `tail -f` (follow — следить).

```bash
tail -f /var/log/syslog
```

### Что делает `tail -f`
- Показывает **последние 10 строк** файла
- Остаётся висеть в терминале
- Новые строки появляются **в реальном времени**

### Полезные вариации
| Команда | Что делает |
|---------|------------|
| `tail -f file` | следить за файлом |
| `tail -n 50 -f file` | показать последние 50 строк и следить |
| `tail -F file` | следить даже если файл пересоздадут |
| `tail -f file1 file2` | следить за несколькими файлами |

```bash
# Примеры 
tail -f /var/log/nginx/access.log
tail -n 20 -f /var/log/syslog
tail -F /var/log/myapp.log   # при ротации логов
```

> **Как выйти:** нажать `Ctrl + C`

---


| Что нужно сделать | Команда |
|-------------------|---------|
| Посмотреть весь маленький файл | `cat file` |
| Постраничный просмотр большого файла | `less file` |
| Первые 10 строк | `head file` |
| Последние 10 строк | `tail file` |
| 30-ю строку файла | `head -n 30 file \| tail -n 1` |
| Найти файл со словом | `grep -rl "слово" /path` |
| Следить за логом в реальном времени | `tail -f /var/log/file` |

---

## Итог

| Команда | Ключи | Назначение |
|---------|-------|------------|
| `cat` | — | вывести весь файл |
| `less` | — | постраничный просмотр |
| `head` | `-n` | первые N строк |
| `tail` | `-n`, `-f`, `-F` | последние N строк / слежение |
| `grep` | `-r`, `-l`, `-i` | рекурсивный поиск по файлам |

## Задание 3: Найти слово `games` в `/etc/passwd` с помощью `less`

```bash
less /etc/passwd
```

**Действия внутри `less`:**
| Команда | Что делает |
|---------|------------|
| `/games` | найти слово `games` (прямой поиск) |
| `n` | перейти к **следующему** совпадению |
| `N` | перейти к **предыдущему** совпадению |
| `q` | выйти из `less` |

## Задание 6: Найти все файлы с текстом `sync` **независимо от регистра**

```bash
grep -rli sync /etc
```

| Ключ | Назначение |
|------|------------|
| `-r` | рекурсивно |
| `-l` | только имена файлов |
| `-i` | **игнорировать регистр** (sync, Sync, SYNC, sYnC) |

```bash
# Сравнение
grep -rl sync /etc     # только "sync"
grep -rli sync /etc    # "sync", "Sync", "SYNC", "sYnC" и т.д.
```

---


| Что нужно сделать | Команда |
|-------------------|---------|
| Первые N строк | `head -n N file` |
| Последние N строк | `tail -n N file` |
| Строки с N-й до конца | `tail -n +N file` |
| Поиск в `less` | `/слово`, `n`, `N`, `q` |
| Найти слово с номером строки | `grep -n слово file` |
| Найти файлы с текстом (рекурсивно) | `grep -rl текст /path` |
| Найти файлы (игнорируя регистр) | `grep -rli текст /path` |

|---------|-------|------------|
| `head` | `-n` | первые N строк |
| `tail` | `-n`, `+N` | последние N строк / с N-й до конца |
| `less` | — | постраничный просмотр + поиск |
| `grep` | `-n`, `-r`, `-l`, `-i` | поиск с номером строки / рекурсивно / имена файлов / игнор регистра |

## Важные выводы
1. `tail -n +14` — показывает **все строки с 14-й до конца**
2. `;` или `&&` — выполняют несколько команд в одной строке
3. В `less` поиск через `/`, переход по `n` и `N`
4. `grep -n` — номер строки
5. `grep -l` — только имена файлов
6. `grep -i` — игнорировать регистр
