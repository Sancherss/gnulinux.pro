# Вопросы и ответы: Перемещение, переименование, удаление. Жёсткие и символические ссылки
---

## 1. Что происходит с данными при перемещении в рамках одной файловой системы и разных файловых систем?

**Ответ:** 

| Ситуация | Что происходит |
|----------|----------------|
| **В рамках одной ФС** | Создаётся новая жёсткая ссылка (в новом месте), старая удаляется. Инод не меняется. Данные остаются на месте. |
| **Между разными ФС** | Данные реально копируются на новую ФС, создаётся новый инод, старые данные удаляются. |

```bash
# Проверка: одинаковый ли инод после mv
ls -i file.txt
# 12345 file.txt

mv file.txt ../other-dir/
ls -i ../other-dir/file.txt
# 12345 file.txt  ← тот же инод (одна ФС)
```

---

## 2. Что такое жёсткая ссылка?

**Ответ:** это, грубо говоря, ещё одно имя файла. Все жёсткие ссылки указывают на одни и те же данные (один инод).

```bash
ln original.txt hardlink.txt
# Теперь original.txt и hardlink.txt — два имени одного файла
```

---

## 3. Что такое символическая ссылка?

**Ответ:** это специальный файл, который хранит путь к другому файлу (или директории).

```bash
ln -s /path/to/target symlink.txt
# symlink.txt -> /path/to/target
```

---

## 4. Удаляет ли файл команда `rm`?

**Ответ:** нет, `rm` удаляет **жёсткую ссылку** на файл. Сами данные удаляются только когда количество жёстких ссылок становится равно нулю.

```bash
# Счётчик ссылок = 2
ls -l file.txt
# -rw-r--r-- 2 user user ...

rm file.txt        # удалили одну ссылку
# Данные остались (есть другая ссылка)

rm hardlink.txt    # удалили последнюю ссылку
# Данные реально удалены с диска
```

---

## 5. Создайте жёсткую и символическую ссылки на файл. Удалите файл. Что произойдёт со ссылками и данными?

**Ответ:** 
- **Жёсткая ссылка** — продолжает работать нормально (данные остались на диске)
- **Символическая ссылка** — становится **битой** (dangling link), показывает, что файл не существует

```bash
# Создаём файл и ссылки
echo "data" > file.txt
ln file.txt hardlink.txt
ln -s file.txt symlink.txt

# Удаляем оригинал
rm file.txt

cat hardlink.txt   # data
cat symlink.txt    # No such file or directory
```

---

## 6. Как отличить в выводе `ls -l` файл, директорию и символическую ссылку?

**Ответ:** по первому символу в выводе `ls -l`:

| Символ | Тип |
|--------|-----|
| `-` | обычный файл |
| `d` | директория |
| `l` | символическая ссылка |

```bash
ls -l
# -rw-r--r-- ... file.txt    ← файл
# drwxr-xr-x ... dir/        ← директория
# lrwxrwxrwx ... link -> file.txt  ← ссылка
```

---

## 7. Как понять, сколько жёстких ссылок у файла?

**Ответ:** использовать `ls -l`. Количество жёстких ссылок отображается **вторым столбцом** вывода.

```bash
ls -l file.txt
# -rw-r--r-- 3 user user 1024 Jan 15 10:00 file.txt
#            ↑
#            количество жёстких ссылок = 3
```

---

## 8. Как понять, являются ли два файла жёсткими ссылками на один файл?

**Ответ:** посмотреть иноды командой `ls -i`. Если иноды **совпадают** — это жёсткие ссылки на один файл.

```bash
ls -i file1.txt file2.txt
# 12345678 file1.txt
# 12345678 file2.txt   ← одинаковый инод → жёсткие ссылки
```

---


| Что нужно узнать | Команда |
|------------------|---------|
| Инод файла | `ls -i file` |
| Количество жёстких ссылок | `ls -l file` (второй столбец) |
| Тип файла (файл/дир/ссылка) | `ls -l` (первый символ) |
| Найти все жёсткие ссылки | `find / -inum XXXXX` |
| Проверить, битая ли ссылка | `test -e symlink && echo "OK" || echo "Broken"` |

---

## Итог

| Понятие | Суть |
|---------|------|
| **Жёсткая ссылка** | Ещё одно имя того же файла (один инод) |
| **Символическая ссылка** | Файл с путём к другому файлу ("ярлык") |
| **`rm`** | Удаляет ссылку, а не данные (пока счётчик > 0) |
| **`mv` в одной ФС** | Переименование ссылки (инод не меняется) |
| **`cp`** | Создаёт новый инод и копирует данные |

## Задание 1: Создание скрытой директории и перемещение файла

```bash
# Создаём файл
touch file

# Создаём скрытую директорию
mkdir .dir

# Перемещаем файл в скрытую директорию
mv file .dir

# Проверяем
ls .dir
# file
```

**Результат:** файл `file` успешно перемещён в скрытую папку `.dir`

---

## Задание 2: Массовое создание, ссылки и удаление

```bash
# 1. Создаём файлы file20..file29 одной командой
touch file{20..29}

# 2. Создаём жёсткие ссылки на эти файлы в скрытой директории .dir
ln file{20..29} .dir/

# 3. Удаляем оригиналы в текущей директории
rm file{20..29}
```

**Проверка:**
```bash
ls .dir/
# file20 file21 file22 file23 file24 file25 file26 file27 file28 file29
```

> Жёсткие ссылки в `.dir` остались, данные сохранены.

## Задание 3: Создание символических ссылок

```bash
# 1. Ссылка на существующий файл
touch file
ln -s file softfile

# 2. Ссылка на существующую директорию
mkdir dirc
ln -s dirc dird

# 3. Ссылка на несуществующий файл
ln -s file1 fake_file

# 4. Ссылка на несуществующую директорию
ln -s dir1 fake_dir
```

**Проверка:**
```bash
ls -l
# lrwxrwxrwx softfile -> file    # работает
# lrwxrwxrwx dird -> dirc         # работает
# lrwxrwxrwx fake_file -> file1   # битая
# lrwxrwxrwx fake_dir -> dir1     # битая
```

## Задание 4: Создание файла и переименование без cd

```bash
# Создаём директорию
mkdir dir1

# Создаём файл file1 внутри dir1 (без cd!)
touch dir1/file1

# Переименовываем file1 → file2 (без cd!)
mv dir1/file1 dir1/file2
```

**Проверка:**
```bash
ls dir1/
# file2
```

## Сводка использованных команд

| Команда | Ключи | Назначение |
|---------|-------|------------|
| `touch` | — | создание пустых файлов |
| `mkdir` | — | создание директории |
| `mv` | — | перемещение / переименование |
| `ln` | (без ключей) | создание жёсткой ссылки |
| `ln` | `-s` | создание символической ссылки |
| `rm` | — | удаление файла (ссылки) |
| `{}` | — | генерация последовательностей (`file{20..29}`) |
