## Вопросы и ответы

### Вопрос 1: Как с помощью if можно проверить несколько условий разом?

**Мой ответ:** `if` а внутри можно использовать `&&` или `||` или `-a` `-o` внутри `[]`

**Правильный ответ:** с помощью операторов `&&` (И) и `||` (ИЛИ) или с помощью `-a` (and) и `-o` (or) внутри `[]`.

| Способ | Пример |
|--------|--------|
| `&&` и `||` | `if [ условие1 ] && [ условие2 ]; then` |
| `-a` и `-o` | `if [ условие1 -a условие2 ]; then` |

---

### Вопрос 2: Как сделать, чтобы команда запустилась если оба условия выполнились? Или выполнилось только одно условие?

**Мой ответ:** `&&` это как логическое И, должно выполняться 1 и 2 условие, `||` это как логическое ИЛИ, хотя бы одно истинно

**Правильный ответ:**

| Оператор | Название | Условие |
|----------|----------|---------|
| `&&` | И (AND) | **оба** условия должны быть истинны |
| `||` | ИЛИ (OR) | хотя бы **одно** условие истинно |

**Таблица истинности:**

| Условие1 | Условие2 | `&&` (И) | `||` (ИЛИ) |
|----------|----------|----------|-------------|
| истина | истина | истина | истина |
| истина | ложь | ложь | истина |
| ложь | истина | ложь | истина |
| ложь | ложь | ложь | ложь |

## Практическое задание

### Условие задания

Создайте скрипт, который:
1. Запрашивает имя пользователя
2. Выводит информацию: UID, домашнюю директорию, список групп
3. Спрашивает, что менять (UID, группу, домашнюю директорию)
4. Если UID — проверить доступность, если нет — один раз предложить ввести заново
5. Если домашнюю директорию — спросить новую директорию и нужно ли перемещать
6. Если группу — спросить, основную или дополнительную
7. Вывести итоговую команду

---

### Мой скрипт

```bash
#!/usr/bin/env bash

read -p "Enter username: " user

# Проверка существования пользователя
if ! id "$user" &>/dev/null; then
    echo "There is no such user"
    exit 1
fi

# Информация о пользователе
uid=$(id -u "$user")
echo "ID пользователя: $uid"
echo "Список групп: $(groups "$user")"
echo "Домашняя директория: $(grep "^$user:" /etc/passwd | cut -d':' -f 6)"

read -p "Что вы хотите поменять? (1-UID, 2-Группа, 3-Домашняя директория): " chg_id

# Смена UID
if [ "$chg_id" = 1 ]; then
    read -p "Введите новый UID: " new_uid
    
    # Проверка, свободен ли UID
    if getent passwd "$new_uid" &>/dev/null; then
        echo "UID $new_uid уже занят"
        read -p "Введите другой UID: " new_uid
    fi
    echo "Итоговая команда: sudo usermod -u $new_uid $user"

# Смена группы
elif [ "$chg_id" = 2 ]; then
    read -p "Меняем основную (1) или дополнительную (2) группу: " chg_gr_id
    
    if [ "$chg_gr_id" = 1 ]; then
        read -p "Введите название основной группы: " groupname
        echo "Итоговая команда: sudo usermod -g $groupname $user"
    elif [ "$chg_gr_id" = 2 ]; then
        read -p "Введите название дополнительной группы: " groupname
        echo "Итоговая команда: sudo usermod -aG $groupname $user"
    else
        echo "Неправильный ввод"
        exit 1
    fi

# Смена домашней директории
elif [ "$chg_id" = 3 ]; then
    read -p "Введите новую домашнюю директорию: " homedir
    read -p "Переместить содержимое старой домашней директории? (y/n): " move_home
    
    if [ "$move_home" = "y" ]; then
        echo "Итоговая команда: sudo usermod -d $homedir -m $user"
    else
        echo "Итоговая команда: sudo usermod -d $homedir $user"
    fi

else
    echo "Неправильный ввод"
    exit 1
fi
```

## Ключевые моменты

### Почему `-aG`, а не `-G`

| Команда | Результат |
|---------|-----------|
| `usermod -G group user` | **заменяет** все дополнительные группы (опасно) |
| `usermod -aG group user` | **добавляет** группу (безопасно) |

### Почему проверка UID через `getent`

```bash
if getent passwd "$new_uid" &>/dev/null; then
    echo "UID занят"
fi
```

- `getent` работает с локальными и сетевыми пользователями (LDAP)
- `&>/dev/null` скрывает вывод, оставляя только проверку кода выхода

### Почему кавычки важны

```bash
# Правильно
if [ "$chg_id" = 1 ]; then

# Неправильно (ошибка, если переменная пуста)

```
