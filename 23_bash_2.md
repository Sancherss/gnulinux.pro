## Bash-скрипты №2 (условия, if, проверки)

## Вопросы и ответы

### Вопрос 1: Какой синтаксис у команды if?

**Мой ответ:** `if then fi`

**Правильный ответ:**

```bash
# Основной синтаксис
if [ условие ]; then
    команды
fi

# С else
if [ условие ]; then
    команды1
else
    команды2
fi

# С elif
if [ условие1 ]; then
    команды1
elif [ условие2 ]; then
    команды2
else
    команды3
fi
```

**Примеры:**
```bash
if [ -f /etc/passwd ]; then
    echo "Файл существует"
fi

if [ "$name" = "root" ]; then
    echo "Привет, админ!"
else
    echo "Привет, $name"
fi
```

>  пробелы после `[` и перед `]` обязательны

### Вопрос 2: Как узнать код выхода предыдущей команды?

**Мой ответ:** `echo $?`

**Правильный ответ:** с помощью специальной переменной `$?`.

| Код | Значение |
|-----|----------|
| `0` | команда выполнена **успешно** |
| `1-255` | команда завершилась с **ошибкой** |

**Примеры:**
```bash
ls /home
echo $?   # 0 — успешно

ls /nonexistent
echo $?   # 2 — ошибка

# Проверка в скрипте
grep "root" /etc/passwd
if [ $? -eq 0 ]; then
    echo "Найдено"
fi

# Сокращённая запись (то же самое)
if grep "root" /etc/passwd; then
    echo "Найдено"
fi
```

>  `$?` показывает код только **последней** выполненной команды.

### Вопрос 3: Как проверить, равно ли значение переменной определённому тексту?

**Мой ответ:** `[]`

**Правильный ответ:** с помощью `[ "$переменная" = "текст" ]`.

```bash
name="root"
if [ "$name" = "root" ]; then
    echo "Это администратор"
fi
```

**Операторы сравнения:**

| Оператор | Значение |
|----------|----------|
| `=` или `==` | равно |
| `!=` | не равно |
| `-z` | строка пустая |
| `-n` | строка не пустая |

```bash
# Проверка на пустоту
if [ -z "$name" ]; then
    echo "Имя не введено"
fi

# Проверка на неравенство
if [ "$name" != "root" ]; then
    echo "Это не root"
fi
```

>  Всегда ставьте пробелы после `[` и перед `]`.

### Вопрос 4: Почему стоит переменную брать в кавычки при использовании `[]`?

**Мой ответ:** потому что в переменную могут передать несколько слов и тогда будет `a b c = it`

**Правильный ответ:** без кавычек переменная может содержать пробелы или быть пустой, что приведёт к синтаксической ошибке.

**Проблема 1: Переменная с пробелами**
```bash
name="John Doe"

# Без кавычек — ошибка!
if [ $name = "John Doe" ]; then   # [ John Doe = John Doe ]
    echo "Равно"
fi
# bash: [: too many arguments

# С кавычками — правильно
if [ "$name" = "John Doe" ]; then
    echo "Равно"
fi
```

**Проблема 2: Пустая переменная**
```bash
name=""

# Без кавычек — ошибка!
if [ $name = "root" ]; then   # [ = root ]
    echo "Равно"
fi
# bash: [: =: unary operator expected

# С кавычками — правильно
if [ "$name" = "root" ]; then
    echo "Равно"
fi
```

**Сравнение:**

| Без кавычек | С кавычками |
|-------------|-------------|
| `[ $name = "text" ]` | `[ "$name" = "text" ]` |
| если `$name` пустое → `[ = text ]` (ошибка) | если `$name` пустое → `[ "" = "text" ]` |
| если `$name` с пробелами → ошибка | если `$name` с пробелами → ок|

## Практические задания

### Задание 1: Проверка, является ли пользователь администратором

**Условие:** скрипт спрашивает имя пользователя и говорит, является ли он администратором.

**Мой скрипт:**
```bash
#!/usr/bin/env bash
read -p "Введите имя пользователя: " username
echo "Является ли пользователь администратором"
if [ $username = root ]
then echo "YES"
else echo "NO"
fi
```

**Улучшенная версия (с проверкой по UID):**
```bash
#!/usr/bin/env bash
read -p "Введите имя пользователя: " username

uid=$(id -u "$username" 2>/dev/null)

if [ "$uid" = "0" ]; then
    echo "Пользователь $username является администратором (UID 0)"
else
    echo "Пользователь $username НЕ является администратором"
fi
```

**Пример работы:**
```
Введите имя пользователя: root
Пользователь root является администратором (UID 0)

Введите имя пользователя: sancher
Пользователь sancher НЕ является администратором
```

### Задание 2: Проверка использования sudo

**Условие:** скрипт проверяет bash историю и логи sudo.

**Мой скрипт:**
```bash
#!/usr/bin/env bash
grep "sudo" ~/.bash_history
if [ $? = 0 ]
then
    echo "YES"
    echo $(grep "sudo" /var/log/secure)
else
    echo "NO"
fi
```

**Улучшенная версия:**
```bash
#!/usr/bin/env bash
history_file="$HOME/.bash_history"
secure_log="/var/log/secure"

if [ ! -f "$history_file" ]; then
    echo "Файл истории не найден: $history_file"
    exit 1
fi

if grep -q "sudo" "$history_file"; then
    echo "YES — пользователь использовал sudo"
    echo "Последние записи из логов sudo:"
    sudo grep "sudo" "$secure_log" | tail -10
else
    echo "NO — пользователь не использовал sudo"
fi
```

**Пример работы:**
```
YES — пользователь использовал sudo
Последние записи из логов sudo:
Apr 21 10:00:01 centos8 sudo: sancher : TTY=pts/0 ; COMMAND=/bin/ls
```

### Задание 3: Проверка имени "John Doe"

**Условие:** скрипт спрашивает имя и фамилию, если "John Doe" — выводит "Access Granted".

**Мой скрипт:**
```bash
#!/usr/bin/env
read -p "Введите имя: " name
read -p "Введите фамилию: " lastname
if [ "$name $lastname" = "John Doe" ]
then
    echo "Access Granted"
fi
```

**Исправленная версия:**
```bash
#!/usr/bin/env bash
read -p "Введите имя: " name
read -p "Введите фамилию: " lastname

if [ "$name $lastname" = "John Doe" ]; then
    echo "Access Granted"
else
    echo "Access Denied"
fi
```

**Пример работы:**
```
Введите имя: John
Введите фамилию: Doe
Access Granted

Введите имя: Jane
Введите фамилию: Smith
Access Denied
```

### Синтаксис if

| Конструкция | Значение |
|-------------|----------|
| `if [ условие ]; then команды; fi` | основная форма |
| `if [ условие ]; then команды1; else команды2; fi` | с else |
| `if [ условие1 ]; then команды1; elif [ условие2 ]; then команды2; else команды3; fi` | с elif |

### Операторы сравнения строк

| Оператор | Значение |
|----------|----------|
| `[ "$a" = "$b" ]` | равно |
| `[ "$a" == "$b" ]` | равно (альтернатива) |
| `[ "$a" != "$b" ]` | не равно |
| `[ -z "$a" ]` | строка пустая |
| `[ -n "$a" ]` | строка не пустая |

### Операторы проверки файлов

| Оператор | Значение |
|----------|----------|
| `[ -f "$file" ]` | файл существует |
| `[ -d "$dir" ]` | директория существует |
| `[ -e "$path" ]` | путь существует |
| `[ -r "$file" ]` | файл доступен для чтения |
| `[ -w "$file" ]` | файл доступен для записи |
| `[ -x "$file" ]` | файл доступен для выполнения |

### Коды выхода

| Конструкция | Значение |
|-------------|----------|
| `$?` | код выхода последней команды |
| `exit 0` | успешное завершение скрипта |
| `exit 1` | завершение с ошибкой |

### Переменные

| Конструкция | Значение |
|-------------|----------|
| `$1, $2, ...` | позиционные параметры (аргументы) |
| `$#` | количество аргументов |
| `$@` | все аргументы |
| `"$var"` | значение переменной (в кавычках) |
