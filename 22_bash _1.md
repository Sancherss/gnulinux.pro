## Полный отчёт по теме: Bash-скрипты №1

## Вопросы и ответы

### Вопрос 1: Для чего нужны скрипты?

**Мой ответ:** скрипты нужны для автоматизации процессов администрирования.

**Правильный ответ:** скрипты нужны для автоматизации повторяющихся задач, выполнения последовательности команд, обработки данных и создания инструментов для администрирования.

| Задача | Пример |
|--------|--------|
| Автоматизация повторяющихся задач | бэкапы, очистка логов |
| Выполнение последовательности команд | установка ПО, настройка сервера |
| Обработка данных | парсинг логов, фильтрация вывода |

---

### Вопрос 2: Что из себя представляет bash скрипт?

**Мой ответ:** bash скрипт представляет из себя набор bash команд.

**Правильный ответ:** bash скрипт — это текстовый файл, содержащий набор команд bash, которые выполняются последовательно.

```bash
#!/bin/bash
# Это комментарий
echo "Hello World"
ls -la
pwd
```

| Элемент | Описание | Пример |
|---------|----------|--------|
| Шебанг | указывает интерпретатор | `#!/bin/bash` |
| Комментарии | пояснения (не выполняются) | `# это комментарий` |
| Команды | обычные bash команды | `ls`, `echo`, `cd` |

> Скрипт — это программа на bash. Всё, что вводишь в терминале, можно записать в скрипт.

---

### Вопрос 3: Для чего нужен шебанг?

**Мой ответ:** шебанг нужен чтобы указать в какой оболочке будет выполняться скрипт.

**Правильный ответ:** шебанг (shebang) — это первая строка скрипта, которая указывает, какой интерпретатор нужно использовать для выполнения скрипта.

| Шебанг | Интерпретатор |
|--------|---------------|
| `#!/bin/bash` | bash |
| `#!/bin/sh` | sh |
| `#!/usr/bin/env bash` | bash (более переносимо) |

| Без шебанга | С шебангом |
|-------------|------------|
| нужно явно вызывать интерпретатор: `bash script.sh` | можно запускать как программу: `./script.sh` |

> Шебанг — обязательная строка в любом скрипте, который будет запускаться как исполняемый файл.

---

### Вопрос 4: Как можно запускать bash скрипты?

**Мой ответ:** баш скрипты можно либо через bash либо через `./`, только для начала надо сделать файл исполняемым.

**Правильный ответ:** есть несколько способов запуска.

| Способ | Команда | Требования |
|--------|---------|------------|
| Через интерпретатор | `bash script.sh` | не нужны права на выполнение |
| Как исполняемый файл | `./script.sh` | нужны права на выполнение (`chmod +x`) |
| Через source | `source script.sh` или `. script.sh` | не нужны права |

```bash
# Сделать скрипт исполняемым
chmod +x script.sh

# Запустить
./script.sh
```

| | `./script.sh` | `source script.sh` |
|--|---------------|---------------------|
| Оболочка | запускает **новый** процесс | выполняется в **текущей** оболочке |
| Переменные | не видны в родительской оболочке | видны в родительской оболочке |

> **Вывод:** `bash script.sh` — самый безопасный способ.

---

### Вопрос 5: Как передать аргументы из командной строки в скрипт?

**Мой ответ:** с помощью `read` можно считать переменные вводимые из терминала либо же через `$1` `$2`.

**Правильный ответ:** есть два основных способа: через **позиционные параметры** (`$1`, `$2`, ...) при вызове скрипта или через **команду `read`** для интерактивного ввода.

| Переменная | Значение |
|------------|----------|
| `$0` | имя скрипта |
| `$1` | первый аргумент |
| `$2` | второй аргумент |
| `$#` | количество аргументов |
| `$@` | все аргументы |

**Пример с позиционными параметрами:**
```bash
#!/bin/bash
echo "Первый аргумент: $1"
```

**Пример с `read`:**
```bash
#!/bin/bash
read -p "Введите имя: " name
echo "Привет, $name!"
```

| | Позиционные параметры | `read` |
|--|----------------------|--------|
| Когда использовать | для передачи аргументов при запуске | для запроса данных во время выполнения |
| Автоматизация | подходит | требует ручного ввода |

> **Вывод:** Позиционные параметры удобны для автоматизации, `read` — для интерактивных скриптов.

---

### Вопрос 6: Как сделать так, чтобы скрипт задавал вопросы и формировал команду?

**Мой ответ:** `echo` выводит вопрос, `read` считывает и там уже проверка на вводимые данные.

**Правильный ответ:** с помощью `echo` (или `read -p`) для вывода вопроса и `read` для считывания ответа, затем подстановка переменной в команду.

```bash
#!/bin/bash
read -p "Введите имя файла: " filename
read -p "Введите права: " perms
touch "$filename"
chmod "$perms" "$filename"
```

## Практические задания

### Задание 1: Поиск пользователя в /etc/passwd

**Условие:** скрипт спрашивает имя пользователя и показывает нужную строчку из `/etc/passwd`.

**Мой скрипт:**
```bash
#!/usr/bin/env bash
read -p "Введите имя пользователя: " name
cat /etc/passwd | grep $name
```

**Пример работы:**
```
Введите имя пользователя: root
root:x:0:0:Super User:/root:/bin/bash
```

**Что использовал:** `read -p`, `cat`, `grep`


---

### Задание 2: Создание файла с заданными правами

**Условие:** скрипт спрашивает название файла и права, создаёт файл и задаёт права.

**Мой скрипт:**
```bash
#!/usr/bin/env bash
read -p "Имя файла: " filename
touch $filename
read -p "Введите права для этого файла: " file_permissions
chmod $file_permissions $filename
```

**Пример работы:**
```
Имя файла: file2
Введите права для этого файла: 777
-rwxrwxrwx. 1 sancher sancher 0 Apr 21 13:43 file2
```

**Что использовал:** `read -p`, `touch`, `chmod`

---

### Задание 3: Вывод оболочки пользователя

**Условие:** скрипт спрашивает имя пользователя и выводит его оболочку.

**Мой скрипт:**
```bash
#!/usr/bin/env bash
read -p "Введите имя пользователя: " username
echo "Оболочка пользователя "$username": "
grep $username /etc/passwd | cut -d':' -f 7
```

**Пример работы:**
```
Введите имя пользователя: root
Оболочка пользователя root: /bin/bash
```

**Что использовал:** `read -p`, `grep`, `cut -d':' -f 7`

### Команды, которые использовал

| Команда | Что делает |
|---------|------------|
| `read -p "текст" var` | вывести текст и прочитать ввод в переменную |
| `echo "текст"` | вывести текст |
| `grep pattern file` | поиск строк по шаблону |
| `cut -d':' -f7` | вырезать 7-е поле с разделителем `:` |
| `touch file` | создать файл |
| `chmod perms file` | изменить права |
| `cat file` | вывести содержимое файла |

### Переменные

| Конструкция | Что делает |
|-------------|------------|
| `name="John"` | присвоить значение |
| `echo $name` | вывести значение |
| `$1`, `$2` | позиционные параметры (аргументы скрипта) |

### Запуск скрипта

| Команда | Что делает |
|---------|------------|
| `bash script.sh` | запуск через интерпретатор |
| `chmod +x script.sh && ./script.sh` | сделать исполняемым и запустить |
| `sudo bash script.sh` | запуск от root |
