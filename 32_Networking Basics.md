## Отчет по сетевым основам

### 1. Ответы на теоретические вопросы

| Вопрос | Ответ |
| :--- | :--- |
| **Обжим витой пары** | Процесс снятия изоляции, расположения жил по стандарту (**T568B** или T568A), вставки в коннектор 8P8C (RJ45) и фиксации кримпером. |
| **MAC-адрес и битность** | Физический адрес сетевой карты. Битность — **48 бит** (записывается как 6 групп шестнадцатеричных цифр). |
| **Условная уникальность MAC** | Да, так как он уникален «с завода», но его можно изменить программно (**spoofing**) или встретить дубликаты у дешевых производителей. |
| **Что такое IP-адрес** | Логический числовой идентификатор устройства в сети, необходимый для передачи данных на сетевом уровне. |
| **Деление на подсети** | Осуществляется с помощью **маски подсети** (Subnet Mask). |
| **Зачем нужен Broadcast** | Для передачи данных **всем** устройствам в текущем сегменте сети (например, поиск DHCP-сервера). |
| **Роутер vs Свитч** | **Свитч** (L2) соединяет устройства внутри одной сети по MAC. **Роутер** (L3) соединяет разные сети по IP. |
| **Передача в другую подсеть** | Компьютер понимает по маске, что цель не в его сети, и отправляет пакет на **шлюз по умолчанию** (Default Gateway). |
| **NAT (проблемы и защита)** | Решает проблему нехватки IPv4-адресов. Защищает, скрывая внутреннюю топологию сети от внешних запросов. |
| **TCP vs UDP** | **TCP** — надежный (с установкой соединения и контролем доставки). **UDP** — быстрый (без гарантий и подтверждений). |
| **Проблема ручных IP** | Решается внедрением протокола **DHCP**, который выдает адреса автоматически. |
| **Преобразование имен** | Используется протокол **DNS** (Domain Name System). |

---

### 2. Практические задания

#### Расчет маски /22
* **Вид x.x.x.x:** `255.255.252.0`
* **Количество адресов:** Всего адресов в подсети — **1024** ($2^{10}$).
    > *Примечание: Для хостов доступно 1022 (минус адрес сети и broadcast).*

#### Примеры протоколов
* **TCP:** HTTP, HTTPS, SSH, FTP, SMTP.
* **UDP:** DNS, DHCP, VoIP (SIP/RTP), TFTP.
