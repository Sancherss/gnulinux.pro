# Типы оболочек (shell), su и окружение

## Вопросы

### Вопрос 1: Типы оболочек (login shell, non-login shell, interactive, non-interactive)

#### Login shell vs Non-login shell

| Тип | Что это | Пример | Какие файлы читает |
|-----|---------|--------|-------------------|
| **Login shell** | требует ввода логина/пароля | SSH, tty (Ctrl+Alt+F3), `su -` | `/etc/profile`, `~/.bash_profile`, `~/.bashrc` |
| **Non-login shell** | не требует ввода логина | эмулятор терминала (иконка), `bash` из командной строки | `~/.bashrc` |

#### Interactive vs Non-interactive shell

| Тип | Что это | Пример |
|-----|---------|--------|
| **Interactive shell** | ты сам вводишь команды | обычный терминал, SSH |
| **Non-interactive shell** | выполняет скрипты | `bash script.sh`, `bash -c "ls"` |

#### Как проверить тип shell

```bash
# Проверить login shell
echo $0
# -bash  (с минусом — login shell)
# bash   (без минуса — non-login)

# Проверить interactive shell
echo $PS1
# если не пусто — interactive
```

### Вопрос 2: Для чего нужна команда su?

**Ответ:** `su` (substitute user) — команда для смены пользователя в текущей сессии.

#### Использование

```bash
# Переключиться на root (требует пароль root)
su

# Переключиться на конкретного пользователя
su sancher

# Переключиться и загрузить окружение целевого пользователя (login shell)
su - sancher
su -l sancher
su --login sancher
```

#### Разница между `su` и `su -`

| Команда | Окружение | Текущая директория | Загружает .bash_profile |
|---------|-----------|---------------------|------------------------|
| `su user` | остаётся от старого пользователя | остаётся | ❌ нет |
| `su - user` | становится как при логине | переходит в $HOME | ✅ да |

#### Пример

```bash
# Я под пользователем sancher в /tmp
sancher@centos8:/tmp$ su root
[root@centos8 tmp]# pwd
/tmp   # директория не изменилась

sancher@centos8:/tmp$ su - root
[root@centos8 ~]# pwd
/root  # перешли в домашнюю директорию root
```

> `su` без аргументов — переключение на **root**.  
> Для выхода из-под другого пользователя: `exit` или `Ctrl+D`.

### Вопрос 3: Чей пароль нужно вводить при использовании su?

**Ответ:** пароль **целевого пользователя**, под которого переключаешься.

#### Правила

| Команда | Какой пароль нужен |
|---------|-------------------|
| `su root` | пароль **root** |
| `su sancher` | пароль **sancher** |
| `su` (без аргументов) | пароль **root** |

#### Особенность для root

```bash
# Если ты уже root
su sancher   # не спрашивает пароль — переключает сразу
```

**Почему?** Root — суперпользователь (UID 0), у которого есть абсолютные права в системе. Root может переключаться на любого пользователя без пароля.

#### Пример

```bash
# Обычный пользователь переключается на другого
sancher@centos8:~$ su vasya
Password: (нужен пароль vasya)

# Root переключается на кого угодно без пароля
[root@centos8 ~]# su sancher
[sancher@centos8 ~]#   # переключился сразу, пароль не спрашивал
```

### Вопрос 4: Файлы формирования окружения

**Ответ:** окружение формируется из нескольких файлов — глобальных и пользовательских.

#### Глобальные файлы (для всех пользователей)

| Файл | Для каких shell | Что содержит |
|------|----------------|--------------|
| `/etc/profile` | login shell | глобальные переменные, PATH |
| `/etc/bashrc` | все Bash | глобальные алиасы, функции |
| `/etc/profile.d/*.sh` | login shell | настройки приложений |
| `/etc/environment` | все (включая GUI) | системные переменные |

#### Пользовательские файлы

| Файл | Для каких shell | Что содержит |
|------|----------------|--------------|
| `~/.bash_profile` | login shell | переменные окружения, экспорт |
| `~/.bashrc` | non-login shell | алиасы, функции, PS1 |
| `~/.profile` | если нет .bash_profile | кроссплатформенный вариант |
| `~/.bash_logout` | при выходе из login shell | очистка, история |

#### Порядок загрузки

**Login shell (SSH, tty, `su -`):**
```
1. /etc/profile
2. /etc/profile.d/*.sh
3. ~/.bash_profile
   ├── если есть ~/.bash_profile → ~/.bashrc (обычно source)
   └── если нет → ~/.bash_login → ~/.profile
```

**Non-login shell (терминал из иконки, `bash`):**
```
1. ~/.bashrc
2. /etc/bashrc (через ~/.bashrc)
```

## Практическое задание

### Задание: Доказать различия между login и non-login shell

#### Эксперимент

**1. Создал переменные в файлах:**

В `~/.bash_profile`:
```bash
test1=test1
```

В `~/.bashrc`:
```bash
test2=test2
```

**2. Проверил non-login shell (`su sancher`):**

```bash
sancher@centos8:/home/testuser$ su sancher
sancher@centos8:/home/testuser$ echo $test1
(пусто)                    # .bash_profile НЕ прочитался

sancher@centos8:/home/testuser$ echo $test2
test2                      # .bashrc прочитался

sancher@centos8:/home/testuser$ echo $0
bash                       # без минуса — non-login shell
```

**3. Проверил login shell (`su - sancher`):**

```bash
sancher@centos8:/home/testuser$ su - sancher
sancher@centos8:~$ echo $test1
test1                      # .bash_profile прочитался

sancher@centos8:~$ echo $test2
test2                      # .bashrc тоже прочитался

sancher@centos8:~$ echo $0
-bash                      # с минусом — login shell
```

#### Результаты

| Тип shell | Команда | `$test1` (.bash_profile) | `$test2` (.bashrc) | `$0` |
|-----------|---------|--------------------------|--------------------|------|
| **non-login** | `su sancher` | пусто | test2 | `bash` |
| **login** | `su - sancher` | test1 | test2 | `-bash` |

#### Вывод

- **Login shell** (`su - user`) читает `~/.bash_profile`
- **Non-login shell** (`su user`) **НЕ** читает `~/.bash_profile`, только `~/.bashrc`
- `$0` показывает тип shell: `-bash` = login, `bash` = non-login

#### Как отличить на практике

```bash
# Проверить тип shell
echo $0
# -bash  → login shell
# bash   → non-login shell

# Или
shopt login_shell
# on  → login shell
# off → non-login shell
```

### Типы shell

| Команда | Тип | Читает .bash_profile |
|---------|-----|---------------------|
| `su user` | non-login | нет |
| `su - user` | login | да |
| `bash` | non-login | нет |
| `bash --login` | login | да |

### Файлы окружения

| Файл | Когда читается |
|------|----------------|
| `/etc/profile` | login shell |
| `/etc/bashrc` | все Bash |
| `~/.bash_profile` | login shell |
| `~/.bashrc` | non-login shell |

### Переключение пользователей

| Команда | Результат |
|---------|-----------|
| `su` | переключение на root (non-login) |
| `su -` | переключение на root (login) |
| `su user` | переключение на user (non-login) |
| `su - user` | переключение на user (login) |
