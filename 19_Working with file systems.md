# Монтирование файловых систем, fstab, UUID, проверка дисков

## Вопросы и ответы

### Вопрос 1: Как увидеть детальную информацию о файловой системе?

**Ответ:** с помощью команд `tune2fs`, `xfs_info`, `dumpe2fs`, `stat`.

| Файловая система | Команда | Пример |
|------------------|---------|--------|
| **ext4** | `sudo tune2fs -l /dev/sda1` | детальная информация |
| **ext4** | `sudo dumpe2fs -h /dev/sda1` | аналогично tune2fs |
| **XFS** | `sudo xfs_info /dev/sda1` | информация о XFS |
| **Любая** | `stat /путь/к/файлу` | информация о блоке, иноде |

#### Ключевые поля для ext4

| Поле | Что означает |
|------|--------------|
| `Filesystem UUID` | уникальный идентификатор ФС |
| `Block size` | размер блока (обычно 4096 байт) |
| `Inode count` | общее количество инод |
| `Free inodes` | свободные иноды |
| `Filesystem state` | состояние (clean = всё хорошо) |

> `tune2fs -l` — команда для диагностики ext4. Для XFS надо использовать `xfs_info`.

---

### Вопрос 2: Зачем нужно монтировать файловые системы?

**Ответ:** чтобы получить доступ к файлам на диске через директорию. Без монтирования файловая система существует, но работать с ней нельзя.

| Без монтирования | После монтирования |
|------------------|---------------------|
| Файловая система есть на диске | Файловая система доступна через директорию |
| Нельзя читать/писать файлы | Можно читать/писать файлы |

#### Пример

```bash
# Файловая система создана, но недоступна
sudo mkfs.ext4 /dev/sdb1

# Примонтировать в директорию
sudo mkdir /mnt/mydisk
sudo mount /dev/sdb1 /mnt/mydisk

# Теперь можно работать с файлами
ls /mnt/mydisk
```

>  Монтирование - это прикрепление файловой системы к определённой директории.

---

### Вопрос 3: Как посмотреть список примонтированных файловых систем?

**Ответ:** с помощью команд `mount`, `df -h`, `findmnt`, `lsblk`.

| Команда | Что показывает | Особенность |
|---------|----------------|-------------|
| `df -h` | точки монтирования + использование места | самый наглядный |
| `mount` | все примонтированные ФС | подробно |
| `findmnt` | дерево монтирования | удобная структура |
| `lsblk` | блочные устройства и точки монтирования | показывает и диски |

#### Пример вывода `df -h`

```bash
df -h
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda3        20G   10G   9G  53% /
/dev/sda1       512M  200M  312M  40% /boot
/dev/sdb1       1.0G  100M  900M  10% /mnt/data
```

> `df -h` — команда для проверки места.

### Вопрос 4: Зачем нужен fstab?

**Ответ:** `fstab` (File Systems Table) — файл, в котором указывается, какие файловые системы и куда монтировать при загрузке системы.

#### Формат записи в fstab

```
<device>  <mountpoint>  <fstype>  <options>  <dump>  <pass>
```

| Поле | Что означает | Пример |
|------|--------------|--------|
| **device** | устройство или UUID | `UUID=...` |
| **mountpoint** | точка монтирования | `/`, `/home` |
| **fstype** | тип ФС | `ext4`, `xfs` |
| **options** | параметры монтирования | `defaults` |
| **dump** | резервное копирование | `0` |
| **pass** | порядок проверки fsck | `1` для корня |

#### Пример /etc/fstab

```bash
UUID=xxx...     /           ext4    defaults    1    1
UUID=yyy...     /boot       ext4    defaults    1    2
/dev/sdb1       /mnt/data   ext4    defaults    0    2
```

> Надо использовать `UUID` вместо `/dev/sdX` в fstab.

### Вопрос 5: Как узнать UUID файловой системы и зачем он нужен?

**Ответ:** UUID можно узнать командой `blkid`, `lsblk -f` или `tune2fs -l`. UUID нужен для **стабильной идентификации** файловой системы.

| Команда | Что показывает |
|---------|----------------|
| `sudo blkid` | UUID всех устройств |
| `sudo blkid /dev/sda1` | UUID конкретного устройства |
| `lsblk -f` | UUID + тип ФС + метка |

#### Почему UUID важен

| Проблема | Решение |
|----------|---------|
| Имена дисков (`/dev/sda`) **меняются** | UUID **всегда одинаковый** |
| В fstab нельзя полагаться на `/dev/sda1` | Используй `UUID=...` |

### Вопрос 6: Как проверить файловую систему на ошибки?

**Ответ:** с помощью команды `fsck` (File System Check) или специализированных утилит.

| Файловая система | Команда |
|------------------|---------|
| **ext4** | `sudo fsck /dev/sda1` |
| **XFS** | `sudo xfs_repair -n /dev/sda1` |

#### Важное правило

> **Файловая система должна быть РАЗМОНТИРОВАНА** перед проверкой.

```bash
# Размонтировать
sudo umount /dev/sda1

# Проверить
sudo fsck /dev/sda1

# Примонтировать обратно
sudo mount /dev/sda1 /mnt
```

#### Коды возврата fsck

| Код | Значение |
|-----|----------|
| 0 | ошибок нет |
| 1 | ошибки исправлены |
| 2 | перезагрузка нужна |
| 4 | ошибки не исправлены |

> Перед проверкой ФС обязательно надо размонтировать её.

## Задание 1: ext4 + автомонтирование

### Условие
Создать директорию `~/myfiles`, скопировать туда `/etc/passwd` и `/etc/group`. Создать раздел с ФС ext4. Перенести данные на эту ФС. Настроить автомонтирование.

### Выполнение

```bash
# 1. Подготовка данных
mkdir ~/myfiles
cp /etc/passwd ~/myfiles/
cp /etc/group ~/myfiles/

# 2. Создание раздела на /dev/sdc (100MB)
sudo fdisk /dev/sdc
# g → GPT таблица
# n → 1 → +100M → w

# 3. Создание ФС ext4
sudo mkfs.ext4 /dev/sdc1

# 4. Перенос данных
sudo mount /dev/sdc1 /mnt
sudo mv ~/myfiles/ /mnt

# 5. Автомонтирование в /etc/fstab
echo "UUID=9a920815-c404-4142-9f0e-201178bf34f3 /home/sancher/myfiles ext4 defaults 0 2" | sudo tee -a /etc/fstab

# 6. Монтирование
mkdir /home/sancher/myfiles
sudo mount /dev/sdc1 /home/sancher/myfiles
```

### Результат

```bash
df -h | grep sdc1
/dev/sdc1       89M   19K   82M   1% /home/sancher/myfiles

lsblk
sdc                          8:32   0    1G  0 disk 
└─sdc1                       8:33   0  100M  0 part /home/sancher/myfiles
```

## Задание 2: XFS + временное монтирование

### Условие
Создать директорию `~/mybackup`. Создать раздел с ФС XFS, временно примонтировать, перенести данные из `~/myfiles`, отмонтировать.

### Выполнение

```bash
# 1. Создание директории
mkdir ~/mybackup

# 2. Создание раздела на /dev/sdc (400MB)
sudo fdisk /dev/sdc
# d → 2 (удалить старый раздел)
# n → 2 → +400M → w

# 3. Создание ФС XFS
sudo mkfs.xfs /dev/sdc2

# 4. Временное монтирование
sudo mount /dev/sdc2 ~/mybackup
systemctl daemon-reload

# 5. Перенос данных
sudo mv ~/myfiles/myfiles/* ~/mybackup/

# 6. Проверка
ls ~/mybackup/
# group  passwd

# 7. Отмонтирование
sudo umount ~/mybackup
```

### Результат

```bash
lsblk
sdc                          8:32   0    1G  0 disk 
├─sdc1                       8:33   0  100M  0 part /home/sancher/myfiles
└─sdc2                       8:34   0  400M  0 part
```

### Монтирование

| Команда | Что делает |
|---------|------------|
| `mount /dev/sdX /mnt` | примонтировать |
| `umount /mnt` | отмонтировать |
| `umount /dev/sdX` | отмонтировать по устройству |
| `mount -a` | примонтировать всё из fstab |

### Просмотр

| Команда | Что делает |
|---------|------------|
| `df -h` | свободное место |
| `df -i` | использование инод |
| `lsblk` | диски и точки монтирования |
| `findmnt` | дерево монтирования |
| `blkid` | UUID устройств |

### fstab

| Команда | Что делает |
|---------|------------|
| `cat /etc/fstab` | просмотреть fstab |
| `nano /etc/fstab` | редактировать fstab |
| `mount -a` | проверить fstab |

### Проверка ФС

| Команда | Что делает |
|---------|------------|
| `sudo fsck /dev/sdX` | проверить ext4 |
| `sudo xfs_repair -n /dev/sdX` | проверить XFS |
| `sudo tune2fs -l /dev/sdX` | информация об ext4 |
| `sudo xfs_info /dev/sdX` | информация о XFS |
