# Файловые системы: ext4, XFS, блоки, сектора, иноды

## Вопросы и ответы

### Вопрос 1: Чем отличается блок файловой системы от сектора?

**Ответ:** сектор — это минимальная **физическая** единица диска, а блок — **логическая** единица, с которой работает файловая система, состоящая из нескольких секторов.

#### Сравнение

| | Сектор | Блок |
|--|--------|------|
| **Что это** | физическая единица диска | логическая единица ФС |
| **Размер** | обычно 512 байт или 4096 байт | кратен секторам (обычно 4 КБ) |
| **Кто определяет** | производитель диска | файловая система |
| **Минимальная операция** | чтение/запись диска | чтение/запись ФС |

#### Связь

```
Блок = несколько секторов

Пример:
Сектор = 512 байт
Блок = 4 КБ = 8 секторов
```

#### Визуализация

```
┌─────────────────────────────────────────────────────────────┐
│                         Блок (4 КБ)                          │
├──────────┬──────────┬──────────┬──────────┬──────────┬───────┤
│ Сектор 1 │ Сектор 2 │ Сектор 3 │ Сектор 4 │ Сектор 5 │ ...   │
│ (512 Б)  │ (512 Б)  │ (512 Б)  │ (512 Б)  │ (512 Б)  │       │
└──────────┴──────────┴──────────┴──────────┴──────────┴───────┘
```

### Вопрос 2: Для чего нужны блоки?

**Ответ:** блоки нужны для эффективной работы с диском. Файл может занимать **несколько блоков**, но минимальная единица — один блок.

#### Основные причины

| Причина | Объяснение |
|---------|------------|
| **Уменьшение количества операций** | читать/писать по 4 КБ выгоднее, чем по 512 байт |
| **Уменьшение накладных расходов** | меньше операций = быстрее работа |
| **Выравнивание с секторами** | блок всегда кратен сектору |

#### Как файл записывается на диск

```
Файл размером 7 КБ при блоке 4 КБ:

┌─────────────────┬─────────────────┬─────────────────┐
│    Блок 1       │    Блок 2       │    Блок 3       │
│    (4 КБ)       │    (3 КБ)       │    (0 КБ)       │
│   файл          │   файл          │   (не занят)    │
└─────────────────┴─────────────────┴─────────────────┘
      ↓                  ↓
  занят полностью    занят частично
```

#### Пустое пространство внутри блока

**Файл занимает ЦЕЛЫЕ блоки.** Если файл меньше блока — остаток пропадает (внутренняя фрагментация).

| Размер файла | Блок 4 КБ | Потеряно |
|--------------|-----------|----------|
| 1 КБ | 4 КБ | 3 КБ |
| 3 КБ | 4 КБ | 1 КБ |
| 4 КБ | 4 КБ | 0 КБ |
| 5 КБ | 8 КБ | 3 КБ |

#### Посмотреть размер блока

```bash
# Размер блока файловой системы
sudo tune2fs -l /dev/sda1 | grep "Block size"

# Или через stat
stat / | grep "Block size"
```

> Блоки уменьшают количество операций ввода-вывода, но приводят к потере места из-за фрагментации.

### Вопрос 3: Чем отличается XFS от ext4?

**Ответ:** главное отличие — **управление инодами**. XFS выделяет иноды динамически, ext4 — фиксированно.

#### Сравнение ext4 и XFS

| Характеристика | ext4 | XFS |
|----------------|------|-----|
| **Иноды** | фиксированное количество при создании ФС | динамические (создаются по мере необходимости) |
| **Максимальный размер ФС** | 1 ЭБ (экзабайт) | 8 ЭБ |
| **Максимальный размер файла** | 16 ТБ | 8 ЭБ |
| **Максимальное количество файлов** | ограничено инодами | практически безлимитно |
| **Дефрагментация** | нужна редко, нет онлайн-дефрагментации | есть онлайн-дефрагментация (`xfs_fsr`) |
| **Уменьшение размера ФС** | можно (`resize2fs`) | **нельзя** |
| **Журналирование** | метаданные + данные (опционально) | только метаданные |
| **Совместимость** | везде (Windows через драйверы) | в основном Linux |

#### Главное: иноды

| ext4 | XFS |
|------|-----|
| Иноды выделяются **при создании ФС** | Иноды выделяются **динамически** |
| Количество инод = размер ФС ÷ байт на иноду | Иноды создаются по мере надобности |
| Можно закончиться (`df -i`) | Нельзя закончиться (практически) |

#### Проверка

```bash
# ext4 — показывает количество инод
df -i /dev/sda1
# Inodes: 655360

# XFS — показывает Inodes = 0 (не лимитировано)
df -i /dev/sdb1
# Inodes: 0
```

#### Когда что выбирать

| Ситуация | Рекомендация |
|----------|--------------|
| Обычный рабочий стол, сервер | ext4 (стабильность, совместимость) |
| Почтовый сервер (много мелких файлов) | **XFS** (динамические иноды) |
| Огромные файлы (видео, бэкапы) | XFS |
| Нужно уменьшить ФС | ext4 (XFS нельзя уменьшить) |

>  ext4 — стандарт для большинства задач. XFS — выбор, когда нужно много файлов или очень большие файлы.

### Вопрос 4: Чем обусловлено максимальное количество файлов в файловой системе?

**Ответ:** максимальное количество файлов ограничено **количеством инод** в файловой системе.

#### Как это работает

| Файловая система | Ограничение | Почему |
|------------------|-------------|--------|
| **ext4** | фиксированное количество инод | иноды выделяются при создании ФС |
| **XFS** | практически нет ограничений | иноды выделяются динамически |

#### Формула для ext4

```
Максимум файлов = количество инод
```

```bash
# Посмотреть максимальное количество файлов
df -i /dev/sda1
# Inodes: 655360 → максимум 655360 файлов и директорий
```

#### Что считается за иноду

| Объект | Занимает иноду? |
|--------|-----------------|
| обычный файл | да |
| директория | да |
| символическая ссылка | да |
| устройство (файл в /dev) | да |
| hard link (жёсткая ссылка) | нет (указывает на существующую иноду) |

#### Что происходит при исчерпании инод

```bash
df -h   # место есть (50% свободно)
df -i   # IUse% = 100%
touch newfile
# touch: cannot touch 'newfile': No space left on device
```

>  Если `df -h` показывает свободное место, а создать файл нельзя —  надо проверить `df -i`. Закончились иноды.

### Вопрос 5: Как посмотреть использование диска?

**Ответ:** для просмотра использования диска используются команды `df`, `du`, `ncdu`, `iostat`.

#### Основные команды

| Команда | Что показывает | Пример |
|---------|----------------|--------|
| `df -h` | использование **разделов** (свободное место) | `df -h` |
| `df -i` | использование **инод** | `df -i` |
| `du -sh` | размер **директории** | `du -sh /home` |
| `du -sh *` | размер всех файлов/папок в текущей директории | `du -sh *` |
| `ncdu` | интерактивный анализ диска (требует установки) | `ncdu /home` |
| `iostat 1` | статистика ввода-вывода дисков в реальном времени | `iostat 1` |

#### Примеры использования

```bash
# Свободное место на всех разделах
df -h

# Свободное место в человекочитаемом формате
df -h /dev/sda1

# Использование инод
df -i

# Размер домашней директории
du -sh /home/

# Размер всех папок в текущей директории (сортировка)
du -sh * | sort -h

# Статистика дисков (обновление каждую секунду)
iostat 1
```

#### Команда iostat (расширенная статистика)

```bash
# Установка
sudo dnf install sysstat   # CentOS/RHEL
sudo apt install sysstat   # Ubuntu

# Использование
iostat 1           # обновление каждую секунду
iostat -x 1        # расширенная статистика
iostat -d sda 1    # только диск sda
```

**Что означают колонки iostat:**

| Поле | Что означает |
|------|--------------|
| `%iowait` | процессор ждёт диск (высокое значение → проблема) |
| `tps` | операций ввода-вывода в секунду |
| `kB_read/s` | чтение в секунду |
| `kB_wrtn/s` | запись в секунду |
| `%util` | загрузка диска (100% = предел) |

#### Сравнение команд

| Команда | Что показывает | Когда использовать |
|---------|----------------|---------------------|
| `df -h` | свободное место на разделах | быстро посмотреть общее состояние |
| `df -i` | количество свободных инод | при ошибке "No space left" |
| `du -sh` | размер папок и файлов | найти, что занимает место |
| `ncdu` | интерактивный анализ | глубокий анализ диска |
| `iostat` | скорость работы диска | при тормозах системы |

>  `df -h` — команда для проверки свободного места. `du -sh *` — чтобы найти, что занимает место. `iostat 1` — чтобы понять, не тормозит ли диск.

### Просмотр использования диска

| Команда | Что делает |
|---------|------------|
| `df -h` | свободное место на разделах |
| `df -i` | использование инод |
| `du -sh /path` | размер директории |
| `du -sh * \| sort -h` | размер всех папок с сортировкой |
| `ncdu /path` | интерактивный анализ (установить) |

### Мониторинг дисков

| Команда | Что делает |
|---------|------------|
| `iostat` | статистика с момента загрузки |
| `iostat 1` | обновление каждую секунду |
| `iostat -x 1` | расширенная статистика |
| `iostat -d sda 1` | только конкретный диск |

### Информация о файловой системе

| Команда | Что делает |
|---------|------------|
| `sudo tune2fs -l /dev/sda1` | информация о ext4 |
| `sudo xfs_info /dev/sda1` | информация о XFS |
| `stat /` | размер блока ФС |
