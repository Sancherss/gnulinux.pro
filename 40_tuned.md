# Оптимизация производительности с tuned

## 1. Введение

**tuned** — демон для автоматической настройки производительности системы.

### Зачем нужен tuned?
- Настройка параметров ядра — сложная задача (требует изучения документации)
- На работе может быть **много систем** и мало времени на ручную настройку
- tuned предоставляет **готовые профили** для разных сценариев
- Позволяет создавать **собственные профили** и централизованно управлять настройками

### Какие параметры настраивает tuned?
- Работа с памятью (например, `vm.swappiness`)
- Энергопотребление процессора (режимы производительности/экономии)
- Скорость работы сетевых адаптеров
- Настройки дисков
- Параметры планировщика процессов
- Другие параметры ядра через sysctl

## 2. Установка и запуск

```bash

```

### Установка (если отсутствует)
```bash
sudo dnf install tuned
sudo systemctl enable --now tuned
```

## 3. Основная утилита — tuned-adm

`tuned-adm` — главная команда для управления tuned.

### 3.1. Основные команды

| Команда | Назначение |
|---------|-----------|
| `tuned-adm list` | Показать все доступные профили |
| `tuned-adm active` | Показать текущий активный профиль |
| `tuned-adm recommend` | Рекомендованный профиль для данной системы |
| `tuned-adm profile <имя>` | Активировать указанный профиль |
| `tuned-adm verify` | Проверить, применился ли профиль корректно |
| `tuned-adm off` | Временно отключить tuned (без остановки сервиса) |

### 3.2. Примеры использования

```bash
# Посмотреть список профилей
tuned-adm list

# Узнать текущий профиль
tuned-adm active

# Активировать профиль для низкой задержки сети
sudo tuned-adm profile network-latency

# Проверить применение профиля
tuned-adm verify

# Временно отключить tuned
tuned-adm off

# Включить обратно (активировать профиль)
tuned-adm profile network-latency
```

## 4. Стандартные профили

### 4.1. Примеры профилей

| Профиль | Назначение |
|---------|-----------|
| `desktop` | Оптимизация для настольных ПК |
| `powersave` | Минимальное энергопотребление |
| `virtual-guest` | Для виртуальных машин (гостевых) |
| `virtual-host` | Для物理ческих серверов с виртуализацией |
| `network-latency` | Низкая задержка в сети |
| `throughput-performance` | Максимальная пропускная способность |
| `latency-performance` | Низкие задержки (основа для других профилей) |

### 4.2. Как выбирается профиль по умолчанию?
- tuned при установке пытается **автоматически определить** оптимальный профиль
- Например, в виртуальной машине выбирается `virtual-guest`

## 5. Директории и файлы профилей

### 5.1. Расположение

| Тип | Директория |
|-----|-----------|
| **Шаблонные профили** | `/usr/lib/tuned/` |
| **Пользовательские профили** | `/etc/tuned/` |

### 5.2. Структура профиля

Каждый профиль — это **директория** с файлом `tuned.conf`:

```
/usr/lib/tuned/network-latency/
└── tuned.conf
```

### 5.3. Пример содержимого tuned.conf

```ini
[main]
summary=Optimize for deterministic performance at the cost of increased power consumption
include=latency-performance

[sysctl]
net.ipv4.tcp_low_latency=1
```

**Секции:**
- `[main]` — основная информация (`summary`, `include`)
- `[sysctl]` — параметры ядра
- Другие плагины (см. ниже)

### 5.4. Просмотр содержимого профиля

```bash
cd /usr/lib/tuned/network-latency
cat tuned.conf

# Посмотреть профиль без комментариев и пустых строк
cd /usr/lib/tuned/powersave
grep -Ev "^#|^$" tuned.conf
```

## 6. Плагины tuned

**Плагины** — механизм, позволяющий единообразно настраивать разные компоненты системы.

| Плагин | Что настраивает |
|--------|----------------|
| `sysctl` | Параметры ядра (через `/proc/sys/`) |
| `bootloader` | Параметры загрузчика GRUB |
| `cpu` | Частота процессора, губернатор |
| `disk` | Настройки дисков (например, `elevator`) |
| `net` | Настройки сети (TCP буферы, очереди) |
| `selinux` | Режим SELinux |
| `script` | Запуск пользовательских скриптов |
| `service` | Управление сервисами (stop/start) |

**Важно:** Каждый плагин имеет свои специфические параметры, единой документации по всем опциям нет.

## 7. Создание собственных профилей

### 7.1. Процесс создания

1. Создать директорию в `/etc/tuned/` с именем профиля
2. Создать внутри файл `tuned.conf`
3. Активировать профиль через `tuned-adm profile`

### 7.2. Возможность наследования

С помощью `include` можно **унаследовать** настройки другого профиля и добавить/переопределить параметры:

```ini
[main]
include=latency-performance

[sysctl]
vm.swappiness=10
```

## 9. Решение проблем с verify

### Что делать, если `tuned-adm verify` показывает ошибку?

1. **Проверить лог:**
   ```bash
   tail /var/log/tuned/tuned.log
   ```

2. **Перезапустить демон:**
   ```bash
   sudo systemctl restart tuned
   ```

3. **Перезагрузить систему** (если параметры требуют перезагрузки)

4. **Проверить конфликты** — ручные настройки могут перебивать параметры tuned

### Пример конфликта

Ситуация: `vm.swappiness` вручную прописан в `/etc/sysctl.d/99-custom.conf`, а tuned требует другое значение.

**Решение:** закомментировать ручные настройки, затем переактивировать профиль:
```bash
sudo tuned-adm profile network-latency
tuned-adm verify
```

## 10. Динамическая оптимизация (dynamic tuning)

### 10.1. Что это?

Обычно профиль применяется **постоянно** и не меняется.

Динамическая оптимизация позволяет **адаптировать параметры** в зависимости от текущей нагрузки:
- При запуске системы — высокая скорость диска
- В обычной работе — сниженное энергопотребление
- Сеть на полную мощность — только при активной передаче данных

### 10.2. Включение

По умолчанию **выключена**. Чтобы включить:

1. Отредактировать `/etc/tuned/tuned-main.conf`:
   ```
   dynamic_tuning=1
   ```

2. Перезапустить tuned:
   ```bash
   sudo systemctl restart tuned
   ```

### 10.3. Ограничения
- Работает не для всех плагинов
- Требует возможности изменения параметров "на лету" (без перезагрузки)
- Больше подходит для **рабочих станций**, чем для серверов


## 13. Ключевые выводы

| Что делает tuned | Какие задачи решает |
|-----------------|---------------------|
| Предоставляет **готовые профили** | Быстрая оптимизация без погружения в детали |
| Позволяет **включать/отключать** профили одной командой | Управление настройками на множестве серверов |
| Поддерживает **наследование** профилей | DRY (Don’t Repeat Yourself) при создании своих профилей |
| Использует **плагины** для разных компонентов | Единый синтаксис для sysctl, GRUB, CPU, дисков, сети |
| Работает в **статическом** и **динамическом** режимах | Гибкость под разные сценарии использования |

## 14. Практические команды 

```bash
# Установка и запуск
sudo dnf install tuned
sudo systemctl enable --now tuned

# Управление профилями
tuned-adm list                    # список всех профилей
tuned-adm active                  # текущий профиль
sudo tuned-adm profile <name>     # активировать профиль
tuned-adm verify                  # проверить применениё

# Отключение/включение
tuned-adm off                     # временно отключить профиль
tuned-adm profile <name>          # снова включить

# Рекомендация
tuned-adm recommend               # что рекомендуют для этой системы

# Диагностика
tail /var/log/tuned/tuned.log     # просмотр логов

# Полное отключение tuned
sudo systemctl stop tuned
sudo systemctl disable tuned
```

## 15. Итог

- **tuned** — мощный инструмент для упрощения настройки производительности
- Подходит как новичкам (готовые профили), так и профессионалам (свои профили с наследованием)
- Централизованное хранение всех параметров в одном месте
- Динамическая оптимизация для рабочих станций
- Интеграция с systemd и sysctl
