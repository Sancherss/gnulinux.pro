# Диски, разделы, MBR и GPT

## Вопросы и ответы

### Вопрос 1: Как посмотреть список подключенных дисков и CD устройств?

**Ответ:** с помощью команд `lsblk`, `fdisk -l`, `parted -l`, `ls /dev/sd*`.

| Команда | Что показывает | Особенность |
|---------|----------------|-------------|
| `lsblk` | дерево всех блочных устройств | самый наглядный |
| `sudo fdisk -l` | список дисков и разделов | требует sudo |
| `sudo parted -l` | список дисков и разделов | более современный |
| `ls /dev/sd*` | список SATA/SCSI дисков | только имена |
| `ls /dev/sr*` | список CD/DVD приводов | только имена |

```bash
# Самый наглядный способ
lsblk

# Подробная информация о дисках
sudo fdisk -l
```

> `lsblk` — показывает дерево дисков и разделов.

---

### Вопрос 2: Почему названия дисков начинаются с sd?

**Ответ:** `sd` означает **SCSI disk** (Small Computer System Interface).

| Аббревиатура | Расшифровка | Что означает |
|--------------|-------------|--------------|
| `sd` | SCSI disk | диски с интерфейсом SCSI |
| `hd` | hard disk | старые IDE диски (устарело) |
| `nvme` | NVMe | современные SSD |

Современные диски (SATA, USB) используют протокол SCSI, поэтому им тоже даются имена `sd*`.

```bash
ls /dev/sd*
# /dev/sda  /dev/sda1  /dev/sdb  /dev/sdb1
```

> `sd` — историческое название, но работает до сих пор. NVMe диски называются `nvme0n1`.

### Вопрос 3: Что такое и чем отличается GPT от MBR?

**Ответ:** MBR и GPT — это таблицы разделов на диске.

#### MBR (Master Boot Record)

| Характеристика | Значение |
|----------------|----------|
| Максимальный размер диска | 2 ТБ |
| Максимальное количество разделов | 4 основных |
| Место хранения | первый сектор диска (512 байт) |
| Резервное копирование | нет |
| Поддержка UEFI | нет (только BIOS) |

#### GPT (GUID Partition Table)

| Характеристика | Значение |
|----------------|----------|
| Максимальный размер диска | 9.4 ЗБ (практически без ограничений) |
| Максимальное количество разделов | 128 (по умолчанию) |
| Место хранения | первый и последний секторы |
| Резервное копирование | есть |
| Поддержка UEFI | да |

#### Сравнение

| Параметр | MBR | GPT |
|----------|-----|-----|
| Макс. размер диска | 2 ТБ | 9.4 ЗБ |
| Макс. количество разделов | 4 | 128 |
| Резервная копия | нет | да |
| Поддержка UEFI | нет | да |
| Контрольная сумма | нет | да |

>  GPT — современный стандарт. MBR нужен только для совместимости со старым BIOS.

### Вопрос 4: Почему в MBR можно создать только 4 раздела и как это можно обойти?

**Ответ:** в MBR для записей о разделах выделено 64 байта. Одна запись занимает 16 байт → 4 раздела.

**Как обойти:** использовать **расширенный (extended) раздел** — контейнер для **логических (logical) разделов**.

```
┌─────────────────────────────────────────────────┐
│  Основной раздел 1 (primary)                    │
├─────────────────────────────────────────────────┤
│  Основной раздел 2 (primary)                    │
├─────────────────────────────────────────────────┤
│  Расширенный раздел (extended) — контейнер      │
│  ┌─────────────────────────────────────────────┐│
│  │  Логический раздел 1 (logical)              ││
│  │  Логический раздел 2 (logical)              ││
│  │  ...                                        ││
│  └─────────────────────────────────────────────┘│
├─────────────────────────────────────────────────┤
│  Основной раздел 4 (primary)                    │
└─────────────────────────────────────────────────┘
```

| Тип | Можно создать | Максимум |
|-----|--------------|----------|
| Primary (основной) | да | 4 |
| Extended (расширенный) | да | 1 |
| Logical (логический) | да (внутри extended) | много |

>  Логические разделы нумеруются с 5 (`sda5`, `sda6`...). На GPT таких проблем нет.

### Вопрос 5: Как посмотреть информацию о дисках?

**Ответ:** с помощью команд `fdisk`, `parted`, `lsblk`.

| Команда | Что показывает |
|---------|----------------|
| `lsblk` | дерево дисков и разделов |
| `sudo fdisk -l` | подробная информация |
| `sudo parted -l` | информация о таблицах разделов |
| `sudo blkid` | UUID и типы ФС |
| `df -h` | использование дисков |

```bash
# Детальная информация о диске
sudo fdisk -l /dev/sda
```

>  `lsblk` — для быстрого просмотра, `sudo fdisk -l` — для детальной информации.

## Практические задания

### Задание 1: Создание разделов (MBR и GPT)

**Условие:** добавить 2 диска по 1ГБ. На первом — MBR и 3 основных раздела по 50МБ. На втором — GPT и 2 раздела по 100МБ.

#### Выполнение

```bash
# Первый диск (MBR)
sudo fdisk /dev/sdb
# o → создать MBR таблицу
# n → p → 1 → +50M
# n → p → 2 → +50M
# n → p → 3 → +50M
# w

# Второй диск (GPT)
sudo fdisk /dev/sdc
# g → создать GPT таблицу
# n → 1 → +100M
# n → 2 → +100M
# w
```

#### Результат

```bash
lsblk
sdb                          8:16   0    1G  0 disk 
├─sdb1                       8:17   0   50M  0 part 
├─sdb2                       8:18   0   50M  0 part 
└─sdb3                       8:19   0   50M  0 part 
sdc                          8:32   0    1G  0 disk 
├─sdc1                       8:33   0  100M  0 part 
└─sdc2                       8:34   0  100M  0 part 
```

| Диск | Таблица | Разделы | Размер | 
|------|---------|---------|--------|
| `/dev/sdb` | MBR | sdb1, sdb2, sdb3 | по 50МБ |
| `/dev/sdc` | GPT | sdc1, sdc2 | по 100МБ |

### Задание 2: Добавление разделов на MBR диск

**Условие:** добавить на первый диск ещё 2 раздела по 100МБ, не удаляя старые.

#### Выполнение

```bash
sudo fdisk /dev/sdb
# n → e (extended) → создать расширенный раздел
# n → логический раздел 5 → +100M
# n → логический раздел 6 → +100M
# w
```

#### Результат

```bash
lsblk
sdb                          8:16   0    1G  0 disk 
├─sdb1                       8:17   0   50M  0 part 
├─sdb2                       8:18   0   50M  0 part 
├─sdb3                       8:19   0   50M  0 part 
├─sdb4                       8:20   0    1K  0 part  # extended (контейнер)
├─sdb5                       8:21   0  100M  0 part  # logical
└─sdb6                       8:22   0  100M  0 part  # logical
```

| Раздел | Тип | Размер |
|--------|-----|--------|
| sdb1 | primary | 50MB |
| sdb2 | primary | 50MB |
| sdb3 | primary | 50MB |
| sdb4 | extended | ~921MB (контейнер) |
| sdb5 | logical | 100MB |
| sdb6 | logical | 100MB |

## Шпаргалка по командам

### Просмотр дисков и разделов

| Команда | Что делает |
|---------|------------|
| `lsblk` | дерево дисков и разделов |
| `sudo fdisk -l` | список всех дисков |
| `sudo fdisk -l /dev/sda` | информация о конкретном диске |
| `sudo parted -l` | информация о таблицах разделов |

### Работа с fdisk

| Команда в fdisk | Что делает |
|-----------------|------------|
| `m` | помощь |
| `p` | показать таблицу разделов |
| `n` | новый раздел |
| `d` | удалить раздел |
| `t` | изменить тип раздела |
| `w` | сохранить и выйти |
| `q` | выйти без сохранения |
| `o` | создать MBR таблицу |
| `g` | создать GPT таблицу |

### Типы разделов в fdisk

| Тип | Обозначение |
|-----|-------------|
| primary | `p` |
| extended | `e` |
| logical | автоматически (после создания extended) |
